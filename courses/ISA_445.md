### Server-side programming

Server-side programming refers to code that runs on a web server rather than in the user's browser, handling tasks such as processing requests, interacting with databases, and generating dynamic content to send back to the client. Most large-scale websites rely on server-side code to dynamically display data pulled from databases, enforce business logic, and manage user sessions. It is the backbone of any interactive web application, complementing the client-side code that handles presentation and user interaction.

- **Languages and Runtimes:** Common server-side languages include Python, JavaScript (Node.js), PHP, Ruby, Java, and C#. Each has mature ecosystems with libraries and tooling for web development.
- **Web Frameworks:** Developers use frameworks such as Django (Python), Express (Node.js), Laravel (PHP), Ruby on Rails, and ASP.NET (C#) to accelerate development by providing routing, templating, and security features out of the box.
- **Request-Response Cycle:** The server receives an HTTP request from the client, processes it (authentication, data retrieval, business logic), and returns an HTTP response containing HTML, JSON, or other data.
- **Middleware:** Intermediate functions that sit between the incoming request and the final route handler, performing tasks like logging, authentication checks, input validation, and error handling.
- **Session and State Management:** Since HTTP is stateless, server-side code manages user sessions through cookies, server-side session stores, or token-based approaches like JSON Web Tokens (JWT).
- **Security Considerations:** Server-side code must guard against common threats including SQL injection, cross-site scripting (XSS), cross-site request forgery (CSRF), and insecure authentication.

**Practical Applications:** Server-side programming powers e-commerce checkout flows, content management systems, and any web application where sensitive data must be processed securely on the server rather than exposed in the browser.

### Web application architecture

Web application architecture defines the structural layout of components that make up a web application, including how the client (browser), server, and database interact with one another. A well-designed architecture separates concerns into distinct layers, improving modularity, maintainability, scalability, and testability. Understanding these architectural patterns is essential for building applications that can grow and adapt to changing requirements.

- **Client-Server Model:** The foundational pattern in which the client (browser or mobile app) sends requests to a server, which processes them and returns responses. All web applications are built on this model.
- **Three-Tier Architecture:** Divides the application into three independent layers: the presentation tier (UI), the application/logic tier (business rules and processing), and the data tier (database storage and retrieval). Each tier can be developed, scaled, and maintained independently.
- **MVC Pattern (Model-View-Controller):** A design pattern where the Model manages data and business logic, the View handles the user interface, and the Controller mediates between the two by receiving user input and coordinating responses. Frameworks like Django, Rails, and ASP.NET MVC implement this pattern.
- **Microservices Architecture:** Decomposes a large application into small, independently deployable services that communicate over APIs. This approach enables teams to develop, deploy, and scale individual services without affecting the entire application.
- **Monolithic vs. Distributed:** Monolithic applications bundle all functionality into a single codebase and deployment unit, which is simpler initially but harder to scale. Distributed architectures (including microservices) trade simplicity for flexibility and horizontal scalability.
- **Load Balancing and Caching:** Production architectures commonly include load balancers to distribute traffic across multiple servers and caching layers (e.g., Redis, CDNs) to reduce database load and improve response times.

**Practical Applications:** E-commerce platforms like Amazon use multi-tier and microservices architectures to independently scale services such as product search, checkout, and recommendations, ensuring high availability even during peak traffic.

### Database connectivity

Database connectivity refers to the mechanisms and protocols that allow a web application's server-side code to communicate with a database management system (DBMS) for storing, retrieving, updating, and deleting data. Establishing efficient and secure database connections is critical for application performance and data integrity. Modern web applications use a combination of connectivity standards, abstraction layers, and pooling strategies to manage database interactions at scale.

- **ODBC and JDBC:** Open Database Connectivity (ODBC) is a language-agnostic standard API for accessing relational databases, while Java Database Connectivity (JDBC) is the Java-specific equivalent. Both provide a uniform interface so that application code can work with different database vendors (MySQL, PostgreSQL, SQL Server) without modification.
- **ORM Frameworks:** Object-Relational Mapping tools such as Hibernate (Java), Entity Framework (.NET), Django ORM (Python), and Sequelize (Node.js) map database tables to programming language objects. ORMs simplify CRUD operations by generating SQL automatically, reducing boilerplate code and the risk of SQL injection.
- **Connection Pooling:** Instead of opening and closing a database connection for every request, connection pools maintain a cache of reusable connections. Libraries like HikariCP (Java) and pg-pool (Node.js) dramatically reduce latency and resource consumption under heavy load.
- **Query Builders and Raw SQL:** Some applications use query builders (e.g., Knex.js) as a middle ground between raw SQL and full ORMs, offering programmatic query construction with more control than an ORM provides.
- **SQL vs. NoSQL:** Relational databases (MySQL, PostgreSQL) use structured schemas and SQL, while NoSQL databases (MongoDB, Redis, Cassandra) offer flexible schemas suited for unstructured or rapidly evolving data models.
- **Security Best Practices:** Parameterized queries and prepared statements prevent SQL injection attacks. Credentials should be stored in environment variables, and connections should use encrypted channels (TLS/SSL).

**Practical Applications:** An online banking application uses connection pooling with parameterized queries to handle thousands of concurrent transactions securely, while an ORM layer simplifies the code needed to map account records to application objects.

### APIs

An Application Programming Interface (API) defines a set of rules and protocols that allow different software systems to communicate with each other. In web development, APIs serve as the bridge between the front-end client and back-end server, as well as between different services and third-party platforms. Understanding API design principles is essential for building interoperable, scalable, and maintainable web applications.

- **REST (Representational State Transfer):** The most widely used API architectural style, REST uses standard HTTP methods (GET, POST, PUT, DELETE) to perform operations on resources identified by URLs. REST APIs are stateless, return data in lightweight formats like JSON, and are well-suited for web and mobile applications.
- **SOAP (Simple Object Access Protocol):** An XML-based messaging protocol with strict standards for security (WS-Security), transactions, and reliability. SOAP is common in enterprise environments such as banking and healthcare where formal contracts (WSDL) and built-in error handling are required.
- **GraphQL:** Developed by Facebook in 2012, GraphQL uses a single endpoint and allows clients to request exactly the data fields they need, avoiding the over-fetching and under-fetching problems common with REST. It is particularly useful for complex data models and mobile applications with bandwidth constraints.
- **API Authentication:** APIs are secured using mechanisms such as API keys, OAuth 2.0 tokens, and JSON Web Tokens (JWT). These ensure that only authorized clients can access protected resources.
- **API Documentation and Testing:** Tools like Swagger/OpenAPI define API contracts in a machine-readable format, while Postman and Insomnia allow developers to test endpoints interactively during development.
- **Rate Limiting and Versioning:** Production APIs implement rate limiting to prevent abuse and versioning (e.g., /api/v1/) to allow backward-compatible evolution of endpoints without breaking existing clients.

**Practical Applications:** A weather application consumes a third-party REST API to fetch forecast data, while a payment system integrates with Stripe's API using OAuth tokens to securely process credit card transactions on behalf of merchants.

### Backend development

Backend development encompasses all server-side logic, infrastructure, and data management that power a web application behind the scenes. It involves writing the code that processes client requests, enforces business rules, manages databases, and returns appropriate responses. While the front end focuses on what users see and interact with, the backend ensures that data flows correctly, securely, and efficiently throughout the entire system.

- **Server Frameworks:** Frameworks like Express.js (Node.js), Django (Python), Spring Boot (Java), and ASP.NET Core (C#) provide structured foundations for building backend applications, offering built-in routing, request handling, and middleware support.
- **Middleware Pipeline:** Middleware functions intercept requests before they reach route handlers, enabling cross-cutting concerns such as logging, authentication, input validation, CORS handling, and error management to be implemented in a clean, reusable manner.
- **Authentication and Authorization:** Backend systems implement authentication (verifying identity) using strategies like session-based auth, JWT tokens, and OAuth 2.0. Authorization (verifying permissions) controls what authenticated users are allowed to access through role-based or policy-based access control.
- **RESTful API Design:** Backend developers design and expose APIs following REST conventions, structuring endpoints around resources, using proper HTTP status codes, and implementing consistent error response formats.
- **Deployment and DevOps:** Modern backend code is deployed using containerization (Docker), orchestration (Kubernetes), and CI/CD pipelines (GitHub Actions, Jenkins). Environment configuration is managed through environment variables and secrets managers.
- **Performance and Scalability:** Techniques such as caching (Redis, Memcached), asynchronous processing (message queues like RabbitMQ or Kafka), horizontal scaling, and database indexing ensure that backend systems can handle increasing traffic and data volumes.

**Practical Applications:** A social media platform's backend handles user registration, authentication via OAuth, stores posts and relationships in a database, serves a REST API consumed by web and mobile clients, and uses message queues to process notifications asynchronously at scale.
