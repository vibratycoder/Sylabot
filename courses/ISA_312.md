### Mobile SDKs

**Definition:** A Software Development Kit (SDK) is a collection of development tools, libraries, APIs, code samples, and documentation that enables developers to build applications for a specific platform such as Android or iOS.

**Key Concepts:**
- **Android SDK:** Provides Android Studio IDE, libraries, APIs, emulators, and code samples for building Android applications
- **iOS SDK:** Offers frameworks, APIs, and resources through Xcode IDE for building apps for iOS, iPadOS, watchOS, and tvOS
- **Cross-Platform SDKs:** Flutter and React Native allow building apps for multiple platforms from a single codebase
- **Key Benefits:** Pre-built UI components, data storage solutions, and platform-specific APIs reduce development time and improve app quality

**Practical Relevance:** Mobile SDKs are essential because each platform (Android, iOS) requires different tools and resources. Understanding SDK selection and integration is critical for efficient mobile development, as SDKs provide the building blocks that let developers focus on application logic rather than low-level platform details.

### Java for mobile

**Definition:** Java is one of the primary programming languages for Android mobile application development, providing a mature ecosystem with object-oriented programming, strong typing, and extensive library support through the Android SDK.

**Key Concepts:**
- **Android Studio:** The official IDE for Android development, providing tools for coding, debugging, testing, and deploying Java-based Android apps
- **Object-Oriented Approach:** Java's OOP paradigm (classes, inheritance, polymorphism) promotes maintainable, reusable code for mobile applications
- **JNI Integration:** Java Native Interface allows seamless integration with C/C++ code for performance-critical tasks
- **Cross-Platform Options:** Frameworks like Codename One enable cross-platform mobile development using a single Java codebase

**Practical Relevance:** Although Kotlin has become Google's preferred language for new Android development, Java remains widely used due to its massive existing codebase, mature tooling, and broad developer community. Learning Java for mobile provides a strong foundation in both programming fundamentals and Android app architecture.

### Mobile web development

**Definition:** Mobile web development involves creating websites and web applications optimized for mobile devices, using responsive design techniques and progressive web app (PWA) technologies to deliver app-like experiences through the browser.

**Key Concepts:**
- **Responsive Web Design (RWD):** Uses flexible layouts, fluid grids, and media queries to adapt content to any screen size; coined by Ethan Marcotte in 2010
- **Progressive Web Apps (PWAs):** Combine web and native app features using service workers for offline support, push notifications, and home screen installation
- **Service Workers:** Act as intermediaries between the network and device, enabling caching and offline functionality
- **PWA Requirements:** A manifest file and a service worker are the minimum requirements for a web app to qualify as a PWA

**Practical Relevance:** Responsive design is the essential foundation for mobile-friendly content, while PWAs transform websites into app-like experiences that bypass app store friction. Together, they allow developers to reach mobile users without building separate native apps, reducing development costs through a single codebase.

### App design

**Definition:** Mobile app design is a user-centered methodology that applies UX/UI principles to create intuitive, accessible, and visually consistent mobile applications through structured research, prototyping, and iterative testing.

**Key Concepts:**
- **Core Principles:** User-centricity and empathy, clarity over complexity (every element should have a purpose), and consistency in color palettes, typography, and interaction patterns
- **Design Process:** User research and interviews, persona creation, information architecture, user flow mapping, wireframing, and usability testing
- **Platform Guidelines:** Apple's Human Interface Guidelines (clarity, simplicity, bottom tab bars) and Google's Material Design (vibrant colors, bold typography, adaptive layouts)
- **Feedback and Affordance:** Every user action should trigger a clear response, and design elements should visually hint at how they are used

**Practical Relevance:** Effective app design directly impacts user retention and satisfaction. Following platform-specific guidelines ensures users feel comfortable with familiar patterns, while user research and iterative testing catch usability problems before launch.

### Mobile UI

**Definition:** Mobile UI (User Interface) design focuses on creating the visual elements and interactive components that users directly engage with on mobile devices, following established design patterns and best practices for touch-based interaction.

**Key Concepts:**
- **Design Patterns:** Reusable solutions to common UI problems, including tab bars (bottom navigation between sections), cards (self-contained info units with images and text), and grid views (scrollable collections of visual content)
- **Best Practices:** Simplicity and consistency, responsive layouts, accessible typography, proper touch target sizing (minimum 44x44 points), and performant animations
- **Navigation Patterns:** Bottom tab bars, hamburger menus, gesture-based navigation, and drawer menus each suit different app structures
- **Pattern Libraries:** Curated collections like Mobbin and UIPatterns.io provide proven, practical design components that maintain consistency and speed up the design process

**Practical Relevance:** Well-designed mobile UI directly impacts usability and user satisfaction. Respecting established UI patterns and platform conventions (iOS vs. Android) reduces the learning curve for users and ensures applications feel native and intuitive on each platform.
