### Big Data

Big Data refers to extremely large and complex datasets -- combining structured, semi-structured, and unstructured data -- that exceed the processing capabilities of traditional database systems. Organizations collect, store, and analyze Big Data to extract valuable insights that drive strategic decision-making. Big Data analytics applies mathematical analysis, optimization, statistical inference, and machine learning to reveal patterns, correlations, and trends hidden within massive datasets.

- **Defining Characteristics**: Big Data is commonly described by the "Three Vs" -- Volume (scale of data), Velocity (speed of data generation and processing), and Variety (diversity of data types and sources). Additional Vs include Veracity (data quality/trustworthiness) and Value (actionable insight derived from the data).
- **Data Sources**: Common sources include social media activity, IoT sensor feeds, clickstream logs, financial transactions, mobile device data, and enterprise system records.
- **Processing Paradigms**: Big Data systems rely on distributed computing frameworks (e.g., Hadoop MapReduce, Apache Spark) that partition work across clusters of commodity hardware to achieve horizontal scalability.
- **Storage Technologies**: Distributed file systems (HDFS), NoSQL databases (HBase, Cassandra, MongoDB), and cloud-based data lakes provide the storage backbone for petabyte-scale datasets.
- **Analytics Techniques**: Descriptive, diagnostic, predictive, and prescriptive analytics are all applied to Big Data, leveraging tools such as machine learning, natural language processing, and graph analysis.
- **Governance and Ethics**: Handling Big Data responsibly requires attention to data privacy, security, regulatory compliance (e.g., GDPR), and ethical use of personal information.

**Practical Applications:** Companies use Big Data analytics for customer segmentation, fraud detection, supply chain optimization, predictive maintenance, and real-time personalization of products and services.

### Hadoop

Apache Hadoop is an open-source software framework designed for distributed storage and processing of large datasets across clusters of commodity hardware. Its architecture is built around two core services -- HDFS (Hadoop Distributed File System) for fault-tolerant storage and MapReduce for parallel data processing. The broader Hadoop ecosystem includes dozens of complementary projects (Hive, Pig, HBase, YARN, Spark, Sqoop, Oozie, Zookeeper) that together provide a comprehensive Big Data platform.

- **HDFS (Hadoop Distributed File System)**: Uses a master/slave architecture with a NameNode (metadata management and namespace) and multiple DataNodes (block-level data storage). Files are split into large blocks (default 128 MB) and replicated across nodes for fault tolerance.
- **MapReduce Programming Model**: Breaks computation into two phases -- a Map phase that filters and sorts data in parallel across nodes, and a Reduce phase that aggregates the intermediate results. This model enables processing of petabyte-scale datasets by distributing work across the cluster.
- **YARN (Yet Another Resource Negotiator)**: Serves as the cluster resource management layer, decoupling resource management from the MapReduce programming model so that multiple processing engines (Spark, Tez, Flink) can share the same cluster.
- **Ecosystem Tools**: Hive provides SQL-like querying over HDFS data; Pig offers a high-level scripting language for data transformations; HBase delivers real-time random read/write access as a NoSQL column-family store; Sqoop handles data transfer between Hadoop and relational databases.
- **Fault Tolerance and Scalability**: Hadoop automatically replicates data blocks (default replication factor of 3) and reschedules failed tasks, enabling reliable operation on inexpensive hardware that may experience individual node failures.

**Practical Applications:** Hadoop is widely deployed for log analysis, ETL pipelines, data warehousing at scale, recommendation engines, and serving as the storage foundation for enterprise data lakes.

### Teradata Aster

Teradata Aster is an analytics platform that combines the accessibility of SQL with the processing power of MapReduce through its patented SQL-MapReduce framework. It enables analysts who know standard SQL to invoke advanced analytics -- such as pattern matching, path analysis, and graph analytics -- without learning separate programming languages. The platform's "Applications-Within" architecture co-locates analytic processing with the data, delivering high performance on large-scale datasets.

- **SQL-MapReduce Framework**: The core innovation of Aster, SQL-MapReduce allows developers to write custom analytic functions in Java, C, C++, C#, Python, or R and expose them as SQL-callable operators. Business analysts can then invoke sophisticated analytics using familiar SQL syntax without writing MapReduce code directly.
- **Pre-Built Analytics Portfolio**: Aster ships with over 50 ready-to-use MapReduce analytic functions covering common Big Data use cases including pattern matching (nPath), sessionization, sentiment analysis, text analytics, marketing attribution, and association analysis.
- **Multi-Genre Analytics**: The platform supports relational (SQL), graph, MapReduce, and statistical analytics in a single engine, eliminating the need to move data between specialized systems for different types of analysis.
- **Discovery Platform**: Aster Discovery Platform provides a visual, interactive environment for data exploration and analysis, enabling data scientists and analysts to iterate quickly on hypotheses against large datasets.
- **Integration with Teradata Ecosystem**: Aster integrates with Teradata's enterprise data warehouse and Hadoop, allowing organizations to combine governed warehouse data with unstructured Big Data sources in a unified analytics workflow.

**Practical Applications:** Organizations use Teradata Aster for customer journey analysis (tracking clickstream paths through websites), fraud pattern detection in financial services, social network analysis, and marketing campaign attribution modeling.

### Social media analytics

Social media analytics is the practice of collecting, measuring, and analyzing data from social media platforms to extract actionable business insights. It encompasses techniques ranging from basic engagement metrics to advanced Natural Language Processing (NLP) for sentiment analysis and opinion mining. Organizations use social media analytics to understand audience behavior, measure brand perception, track campaign performance, and detect emerging trends or crises in real time.

- **Sentiment Analysis**: Uses NLP and machine learning to classify social media text as positive, negative, or neutral. Advanced models detect granular emotions (joy, anger, frustration, surprise) and assign numerical sentiment scores that can be aggregated and tracked over time.
- **Core Metrics and KPIs**: Key measurements include engagement rate (likes, shares, comments), reach and impressions, mention volume, share of voice, follower growth, and sentiment distribution. Sudden spikes in negative mentions serve as early warning signals for reputational crises.
- **Text and Content Analytics**: Techniques include topic modeling, keyword extraction, hashtag analysis, and named entity recognition to identify what audiences are discussing and which themes resonate most.
- **Network and Influence Analysis**: Maps relationships between users, identifies influential accounts and communities, and tracks how content and sentiment propagate through social networks.
- **Multi-Modal Analysis**: Modern social media analytics goes beyond text to analyze images, video, and audio content, incorporating temporal dynamics and interaction patterns for richer insight.
- **Visualization and Dashboards**: Sentiment scores, trend lines, and engagement data are compiled into interactive dashboards that help stakeholders quickly identify patterns, anomalies, and shifts in brand perception.

**Practical Applications:** Brands use social media analytics for real-time campaign monitoring, competitive benchmarking, customer experience improvement, product feedback analysis, and crisis management by detecting and responding to negative sentiment before it escalates.

### Link analysis

Link analysis is a data analysis technique that examines the relationships (links) between nodes in a network to determine the relative importance, influence, or role of each node. Originally developed for ranking web pages, it is grounded in graph theory and models entities as nodes and their connections as edges. The most well-known link analysis algorithm is Google's PageRank, which assigns importance scores to nodes based on the quantity and quality of incoming links.

- **PageRank Algorithm**: Assigns each node a numerical score based on the structure of incoming links. It simulates a "random surfer" who follows outgoing links with probability (1 - d) and jumps to a random node with damping probability d (typically 0.85). Nodes that receive links from other high-scoring nodes receive higher PageRank scores.
- **HITS Algorithm (Hubs and Authorities)**: An alternative to PageRank developed by Jon Kleinberg. It identifies two types of important nodes -- Hubs (nodes that link to many authoritative sources) and Authorities (nodes that are linked to by many hubs) -- through an iterative mutual reinforcement process.
- **Centrality Measures**: Link analysis employs various centrality metrics including degree centrality (number of connections), betweenness centrality (how often a node lies on shortest paths), closeness centrality (average distance to all other nodes), and eigenvector centrality (influence based on connections to other influential nodes).
- **Community Detection**: Algorithms such as modularity optimization and label propagation identify clusters of densely connected nodes within larger networks, revealing natural groupings or communities.
- **Graph Databases and Tools**: Platforms like Neo4j, NetworkX (Python), and Gephi provide storage, querying, and visualization capabilities for link analysis at scale.

**Practical Applications:** Link analysis is used in web search ranking, fraud detection (identifying suspicious transaction networks), social network analysis, cybersecurity threat intelligence, law enforcement investigations, and recommendation systems that leverage user-item interaction graphs.

### Stream analytics

Stream analytics (also called stream processing) is the continuous ingestion, processing, and analysis of data in real time as it is generated, rather than storing it first and analyzing it later in batches. This paradigm enables organizations to detect patterns, trigger alerts, and make decisions within milliseconds to seconds of data arrival. Key frameworks in the stream analytics ecosystem include Apache Kafka (distributed event streaming), Apache Spark Structured Streaming, Apache Flink, and Apache Storm.

- **Apache Kafka**: An open-source distributed event streaming platform capable of handling millions of messages per second with high throughput, fault tolerance, and horizontal scalability. Kafka serves as the central data pipeline, ingesting streams from producers and delivering them to consumer applications. Kafka Streams provides a lightweight library for building stream processing applications directly on top of Kafka.
- **Apache Spark Structured Streaming**: Treats streaming data as an unbounded table, allowing developers to write queries using the same DataFrame/Dataset API used for batch processing. This unified model simplifies development and enables incremental processing with exactly-once fault-tolerance guarantees.
- **Apache Flink**: A distributed stream processing framework designed for stateful computations over unbounded data streams, offering true event-time processing, low latency, and strong consistency guarantees.
- **Windowing and State Management**: Stream processing engines use windowing strategies (tumbling, sliding, session windows) to group events for aggregation. Stateful processing maintains intermediate results across events, enabling complex computations like running averages and sessionization.
- **Event Time vs. Processing Time**: Distinguishing between when an event occurred (event time) and when it is processed (processing time) is critical for handling out-of-order data and producing accurate results, especially with late-arriving events.

**Practical Applications:** Stream analytics powers real-time fraud detection in financial transactions, IoT sensor monitoring and predictive maintenance, live dashboards for operational intelligence, real-time recommendation engines, and continuous monitoring of server logs for anomaly detection.

### Volume/Velocity/Variety

Volume, Velocity, and Variety -- known as the "Three Vs" -- are the foundational dimensions that define Big Data, first articulated by Gartner analyst Doug Laney in a 2001 research publication. These three characteristics distinguish Big Data from conventional datasets and dictate the architectural and analytical approaches required to handle it. Understanding the Three Vs is essential for designing systems that can store, process, and derive value from data at scale.

- **Volume**: Refers to the sheer scale of data generated and collected. Global data creation reached approximately 181 zettabytes in 2025, driven by sources such as social media posts, IoT sensors, transaction logs, and clickstream data. Traditional relational databases cannot efficiently store or query data at this scale, necessitating distributed systems like Hadoop HDFS and cloud data lakes.
- **Velocity**: Describes the speed at which new data is generated, ingested, and must be processed. Social media platforms, financial markets, and IoT devices produce data in real time or near real time. Systems must process this high-velocity data with minimal latency to enable timely decision-making -- stream processing frameworks (Kafka, Spark Streaming, Flink) address this need.
- **Variety**: Captures the diversity of data types and formats. Big Data includes structured data (relational tables, spreadsheets), semi-structured data (JSON, XML, log files), and unstructured data (text, images, audio, video). Handling variety requires flexible storage (NoSQL databases, data lakes) and processing tools capable of parsing and integrating heterogeneous data sources.
- **Extended Vs**: The original three have been expanded by practitioners to include Veracity (data accuracy, trustworthiness, and quality), Value (the business insight and ROI extractable from the data), Variability (inconsistency in data flow rates and meaning), and Visualization (the ability to present complex data in comprehensible formats).

**Practical Applications:** The Three Vs framework guides architectural decisions -- Volume drives storage strategy (distributed file systems vs. traditional databases), Velocity determines the processing paradigm (batch vs. stream), and Variety influences data modeling choices (schema-on-read vs. schema-on-write) and tool selection.
