### Predictive analytics

Predictive analytics is a branch of advanced analytics that uses historical data combined with statistical modeling, data mining techniques, and machine learning to make predictions about future outcomes. It goes beyond descriptive analytics (knowing what happened) to provide a best assessment of what will happen next, enabling proactive decision-making. Organizations across industries rely on predictive analytics to reduce risk, optimize operations, and gain competitive advantage.

- **Core Techniques**: Includes regression models (linear and logistic), decision trees, neural networks, time series models (ARIMA, ARMA), and ensemble methods such as random forests and gradient boosting
- **Process Workflow**: Follows a structured pipeline of data collection, preprocessing and cleaning, exploratory analysis, model building, validation, and deployment into production systems
- **Key Metrics**: Model performance is evaluated using accuracy, precision, recall, F1-score, AUC-ROC for classification, and RMSE, MAE, and R-squared for regression tasks
- **Relationship to Data Mining**: Data mining uncovers hidden patterns in historical data; predictive analytics leverages those patterns to forecast future events and trends
- **Industry Applications**: Used in healthcare (disease prediction, patient readmission), finance (credit scoring, fraud detection), marketing (customer churn, demand forecasting), and supply chain (inventory optimization)
- **Tools and Platforms**: Commonly implemented using Python (scikit-learn, statsmodels), R, SAS, SPSS, and cloud-based platforms like AWS SageMaker and Google AutoML

**Practical Applications**: Retailers use predictive analytics to forecast seasonal demand and optimize inventory levels, while banks deploy predictive models to assess loan default risk and detect fraudulent transactions in real time.

### Regression

Regression analysis is a fundamental statistical method used to estimate the relationship between a dependent variable (outcome) and one or more independent variables (predictors). It serves as the foundation of predictive analytics, allowing analysts to quantify how changes in input variables affect an outcome and to forecast future values. Regression is one of the most widely used techniques in both traditional statistics and modern machine learning.

- **Linear Regression**: Models a straight-line relationship between predictors and a continuous outcome using ordinary least squares (OLS) to minimize the sum of squared residuals; extended to multiple linear regression when there are several predictors
- **Logistic Regression**: Used for binary classification problems where the outcome is categorical (e.g., yes/no, pass/fail); outputs probabilities via the sigmoid function and applies a decision threshold
- **Polynomial and Nonlinear Regression**: Captures curved relationships by adding polynomial terms or using nonlinear functions, allowing models to fit more complex patterns in the data
- **Regularization Techniques**: Ridge regression (L2 penalty) and Lasso regression (L1 penalty) address overfitting and multicollinearity by shrinking or eliminating coefficients; Elastic Net combines both approaches
- **Model Evaluation Metrics**: Performance is assessed using R-squared (proportion of variance explained), adjusted R-squared, Root Mean Squared Error (RMSE), Mean Absolute Error (MAE), and residual analysis
- **Assumptions and Diagnostics**: Classical linear regression assumes linearity, independence of errors, homoscedasticity (constant variance), and normality of residuals; violations require transformations or alternative methods

**Practical Applications**: Businesses use regression analysis to forecast sales revenue based on advertising spend and economic indicators, while real estate professionals build regression models to estimate property values based on features like square footage, location, and number of bedrooms.

### Decision trees

Decision trees are a non-parametric supervised learning method used for both classification and regression tasks. They work by recursively splitting data into subsets based on feature values, creating a hierarchical tree structure where each internal node represents a test on a feature, each branch represents an outcome of that test, and each leaf node holds a predicted class label or value. Their visual, flowchart-like structure makes them one of the most interpretable machine learning algorithms.

- **Tree Structure**: Composed of a root node (the first split), internal/decision nodes (feature tests), branches (outcomes of tests), and leaf/terminal nodes (final predictions); data flows from root to leaf during prediction
- **Splitting Criteria**: Classification trees use metrics like Gini impurity and information gain (entropy) to determine the best feature split; regression trees use variance reduction or mean squared error to evaluate splits
- **Key Algorithms**: ID3 (Iterative Dichotomiser 3) uses information gain, C4.5 improves upon ID3 with gain ratio and handles continuous attributes, and CART (Classification and Regression Trees) produces binary splits using Gini impurity
- **Pruning Techniques**: Pre-pruning (early stopping) limits tree depth or minimum samples per leaf during construction; post-pruning (cost-complexity pruning) removes branches after the full tree is built to reduce overfitting
- **Advantages**: Highly interpretable and easy to visualize, requires minimal data preprocessing (no need for feature scaling or normalization), handles both numerical and categorical data, and captures nonlinear relationships
- **Limitations**: Prone to overfitting when trees grow too deep, sensitive to small changes in data (high variance), and can produce biased trees if classes are imbalanced

**Practical Applications**: Healthcare providers use decision trees to support clinical diagnosis by mapping patient symptoms and test results to probable conditions, while banks use them to automate loan approval decisions based on applicant attributes like income, credit score, and employment history.

### Random forests

Random forests are an ensemble learning method that builds a large collection of decision trees during training and aggregates their predictions to produce a more accurate and robust result. By combining the outputs of many individually trained trees, random forests reduce the overfitting problem common to single decision trees while maintaining strong predictive performance. The algorithm was formalized by Leo Breiman in 2001 and remains one of the most widely used machine learning methods.

- **Bootstrap Aggregating (Bagging)**: Each tree in the forest is trained on a random bootstrap sample (drawn with replacement) from the original training data, ensuring diversity among the individual trees
- **Feature Randomness**: At each node split, only a random subset of features is considered rather than all available features; this decorrelates the trees and further reduces variance in the ensemble
- **Prediction Aggregation**: For classification tasks, the forest uses majority voting across all trees to determine the final class; for regression tasks, the final prediction is the average of all individual tree predictions
- **Out-of-Bag (OOB) Estimation**: Since each tree is trained on a bootstrap sample, approximately one-third of observations are left out and can be used as a built-in validation set to estimate generalization error without a separate test set
- **Feature Importance**: Random forests provide a natural measure of feature importance by tracking how much each feature reduces impurity across all trees, aiding in variable selection and model interpretation
- **Hyperparameters**: Key tuning parameters include the number of trees (n_estimators), maximum tree depth, minimum samples per split, and the number of features considered at each split (max_features)

**Practical Applications**: Random forests are widely used in finance for credit risk modeling and fraud detection, in bioinformatics for gene expression classification, and in e-commerce for recommendation systems and customer segmentation.

### Association analysis

Association analysis (also called association rule mining) is an unsupervised data mining technique that discovers interesting relationships, patterns, and co-occurrences among items in large transactional datasets. The most well-known application is market basket analysis, which identifies products that are frequently purchased together. Introduced by Agrawal and Srikant in 1994, the Apriori algorithm remains the foundational method for extracting association rules.

- **Key Metrics**: Three primary measures evaluate association rules: *support* (how frequently an itemset appears in the dataset), *confidence* (the probability that item B is purchased when item A is purchased), and *lift* (the ratio of observed co-occurrence to expected co-occurrence if items were independent; lift greater than 1 indicates a positive association)
- **Apriori Algorithm**: Uses a level-wise, breadth-first search strategy that generates candidate itemsets of increasing size, pruning those that fall below the minimum support threshold; relies on the "apriori principle" that all subsets of a frequent itemset must also be frequent
- **FP-Growth Algorithm**: An alternative to Apriori that avoids expensive candidate generation by compressing the dataset into a Frequent Pattern tree (FP-tree) and mining patterns directly from this compact structure, offering better performance on large datasets
- **Rule Generation**: After finding frequent itemsets, association rules of the form {A} -> {B} are generated and filtered by minimum confidence and lift thresholds to retain only meaningful and actionable rules
- **Limitations**: Can produce a very large number of rules, many of which may be trivial or redundant; computational cost grows exponentially with the number of items; requires careful threshold tuning to balance rule quantity and quality
- **Beyond Market Basket**: Also applied to web usage mining (page navigation patterns), medical diagnosis (symptom-disease associations), and bioinformatics (gene co-expression patterns)

**Practical Applications**: Retailers use association analysis to design store layouts by placing frequently co-purchased items near each other, create targeted product bundle promotions, and power online recommendation engines ("customers who bought X also bought Y").

### Clustering

Clustering is an unsupervised machine learning technique that groups data points into clusters such that objects within the same cluster are more similar to each other than to those in other clusters. Unlike supervised methods, clustering does not require labeled data and instead discovers natural groupings and hidden structures within datasets. It is one of the most widely used exploratory data analysis techniques in data mining.

- **K-Means Clustering**: Partitions data into K pre-specified clusters by iteratively assigning each point to the nearest centroid and recalculating centroids until convergence; fast and scalable but requires choosing K in advance and assumes spherical, equally-sized clusters
- **Hierarchical Clustering**: Builds a tree-like structure (dendrogram) of nested clusters; agglomerative (bottom-up) starts with each point as its own cluster and merges the closest pairs, while divisive (top-down) starts with one cluster and recursively splits; does not require pre-specifying K
- **DBSCAN (Density-Based Spatial Clustering)**: Groups together points that are closely packed in high-density regions and marks points in low-density regions as outliers; does not require specifying the number of clusters and can find arbitrarily shaped clusters
- **Evaluation Metrics**: Internal metrics include the silhouette score (measures how similar a point is to its own cluster vs. neighboring clusters) and the elbow method (plots within-cluster sum of squares against K); external metrics like the Rand index compare clusters to known labels
- **Distance Measures**: Clustering depends on the choice of distance metric; common options include Euclidean distance, Manhattan distance, cosine similarity, and Mahalanobis distance, each suited to different data types and distributions
- **Preprocessing Requirements**: Feature scaling (standardization or normalization) is critical because most distance-based algorithms are sensitive to variable magnitudes; dimensionality reduction via PCA is often applied before clustering high-dimensional data

**Practical Applications**: Marketing teams use clustering for customer segmentation to tailor campaigns to distinct behavioral groups, while genomics researchers cluster gene expression profiles to identify subtypes of diseases and discover potential biomarkers.

### Python

Python is the dominant programming language for data science, predictive analytics, and machine learning, valued for its readable syntax, extensive ecosystem of libraries, and strong community support. In the context of data mining and predictive analytics (as covered in AA 645), Python provides an end-to-end workflow from data ingestion and cleaning through model building, evaluation, and visualization. Its open-source nature and cross-platform compatibility make it accessible for both academic learning and industry practice.

- **Pandas**: The primary library for data manipulation and analysis; provides DataFrame and Series data structures for loading, cleaning, transforming, filtering, and aggregating structured data from CSV, Excel, SQL, and other sources
- **NumPy**: The foundational numerical computing library; provides efficient multi-dimensional array operations, linear algebra routines, and mathematical functions that underpin nearly all other data science libraries
- **Scikit-learn (sklearn)**: The most widely used machine learning library; offers a consistent API for regression, classification, clustering, dimensionality reduction, model selection (cross-validation, grid search), and preprocessing (scaling, encoding, imputation)
- **Matplotlib and Seaborn**: Core visualization libraries; Matplotlib provides low-level plotting control for line charts, bar charts, scatter plots, and histograms, while Seaborn builds on it with attractive statistical visualizations like heatmaps, pair plots, and violin plots
- **Statsmodels**: Provides classical statistical models including OLS regression, logistic regression, time series analysis (ARIMA), and hypothesis testing with detailed summary output tables familiar to statisticians
- **Jupyter Notebooks**: The standard interactive development environment for data science; allows mixing executable Python code, visualizations, and markdown text in a single document, facilitating reproducible analysis and presentation

**Practical Applications**: A typical predictive analytics workflow in Python involves loading data with Pandas, exploring it with Seaborn visualizations, building a random forest classifier with scikit-learn, evaluating performance with cross-validation, and presenting findings in a Jupyter Notebook.

### Tableau

Tableau is a leading business intelligence and data visualization platform that enables users to create interactive dashboards, reports, and charts through an intuitive drag-and-drop interface. Acquired by Salesforce in 2019, it allows users to connect to virtually any data source and transform raw data into visual insights without requiring programming skills. In the context of data mining and predictive analytics, Tableau serves as the primary tool for communicating analytical findings to stakeholders.

- **Data Connectivity**: Connects to a wide range of data sources including spreadsheets (Excel, CSV), relational databases (SQL Server, MySQL, PostgreSQL), cloud platforms (Google BigQuery, Amazon Redshift, Snowflake), and web data connectors
- **Visualization Types**: Supports a rich library of chart types including bar charts, line graphs, scatter plots, heat maps, tree maps, geographic maps, box plots, bullet graphs, and combination charts; users can also build custom visualizations
- **Interactive Dashboards**: Multiple visualizations can be combined into a single interactive dashboard with filters, parameters, and actions that allow end users to drill down, highlight, and explore data dynamically
- **Calculated Fields and Analytics**: Provides built-in analytics capabilities including trend lines, reference lines, forecasting, clustering, and the ability to create custom calculated fields using Tableau's formula language
- **Product Family**: Tableau Desktop is for authoring visualizations; Tableau Server and Tableau Cloud allow publishing and sharing across organizations; Tableau Prep handles data cleaning and transformation; Tableau Public offers free publishing for public datasets
- **Integration with Python and R**: Through TabPy (Tableau Python Server) and RServe, users can embed Python or R scripts directly into Tableau calculated fields, enabling advanced statistical analysis and machine learning model scoring within dashboards

**Practical Applications**: Business analysts use Tableau to build executive dashboards that monitor KPIs in real time, while data scientists use it to present the results of predictive models through interactive visualizations that allow non-technical stakeholders to explore scenarios and understand model outputs.
