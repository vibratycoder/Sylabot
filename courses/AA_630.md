### Data acquisition

Data acquisition is the process of collecting and importing data from diverse sources, transforming real-world information into digital formats suitable for storage, analysis, and modeling. It encompasses both primary data collection (direct gathering through experiments, surveys, sensors) and secondary data sourcing (APIs, existing databases, third-party providers). The quality and scope of acquired data fundamentally determines the value of all downstream analytical work.

- **Collection Methods:** APIs (standardized programmatic access via REST or GraphQL), web scraping (using Python tools like BeautifulSoup, Scrapy, and Selenium), IoT sensors, database queries, file imports (CSV, JSON, XML, Parquet), and third-party data providers or marketplaces.
- **Acquisition Strategies:** Organizations acquire data by (1) collecting new data directly, (2) converting or transforming legacy data from older systems, (3) sharing or exchanging data with partners, or (4) purchasing data from commercial vendors.
- **Primary vs. Secondary Data:** Primary data is collected firsthand through experiments, surveys, or direct observation, while secondary data comes from pre-existing sources such as published datasets, government repositories, or commercial databases.
- **Approaches:** Continuous acquisition (real-time streaming and monitoring), event-driven collection (triggered by specific conditions), multi-channel integration (simultaneous sources), and batch processing (periodic scheduled extraction).
- **Key Considerations:** Data quality assessment, source reliability and provenance, format compatibility, legal and ethical compliance (privacy regulations like GDPR), licensing terms, and volume/velocity requirements.
- **Planning and Requirements:** Effective acquisition begins with clearly defining analytical objectives, specifying data requirements (schema, granularity, frequency), evaluating source cost and reliability, and building validation checks into the ingestion process.

**Practical Applications:** In a graduate analytics context, a student might use Python's `requests` library to pull data from a public REST API, write SQL queries to extract records from a relational database, and read CSV files from government open-data portals, combining all three sources into a unified pandas DataFrame for analysis.

### Data preparation

Data preparation (also called data preprocessing) is the process of cleaning, transforming, and organizing raw data into a format suitable for analysis and modeling. It is widely recognized as the most time-consuming phase of the analytics workflow, often consuming 60-80% of project effort, yet it has the most significant impact on model accuracy and analytical results. Proper data preparation bridges the gap between raw collected data and the structured inputs that analytical models require.

- **Data Profiling and Discovery:** Before transforming data, analysts examine its structure, distributions, completeness, and relationships using summary statistics, data type checks, and visualization to identify quality issues and inform preparation strategy.
- **Handling Missing Values:** Strategies include deletion (removing incomplete records), imputation (replacing gaps with mean, median, mode, or model-predicted values), and interpolation for time-series data. The choice depends on the proportion and pattern of missingness.
- **Encoding and Scaling:** Categorical variables must be converted to numerical representations (one-hot encoding, label encoding, ordinal encoding) and continuous features may need normalization (min-max scaling) or standardization (z-score) to ensure algorithms treat all features appropriately.
- **Feature Engineering:** Creating new variables from existing data through aggregation, binning, polynomial features, interaction terms, or domain-specific transformations to improve model performance and interpretability.
- **Data Splitting:** Dividing prepared data into training, validation, and test sets to enable unbiased model evaluation, typically using ratios like 70/15/15 or k-fold cross-validation strategies.
- **Tools and Workflow:** Python (pandas for manipulation, NumPy for numerical operations, scikit-learn for preprocessing pipelines), SQL for database-level transformations, and Jupyter Notebooks for interactive, documented preparation workflows.

**Practical Applications:** The quality of analytical results is directly tied to data preparation quality. A typical workflow involves profiling a dataset in pandas, imputing missing values, encoding categorical columns with scikit-learn's `OneHotEncoder`, scaling numerical features with `StandardScaler`, and splitting the result into train/test sets -- all documented in a reproducible notebook.

### Python for analytics

Python is one of the most widely used programming languages for data analytics and data science, offering a rich ecosystem of libraries for data manipulation, statistical analysis, visualization, and machine learning. Its readable syntax and extensive community support make it accessible for analysts while remaining powerful enough for production-grade applications. Python's versatility allows it to serve every stage of the analytics pipeline, from data ingestion to model deployment.

- **pandas:** The primary library for data manipulation, providing DataFrame and Series data structures for loading, filtering, merging, grouping, pivoting, and reshaping tabular data. Key functions include `read_csv()`, `merge()`, `groupby()`, `pivot_table()`, and comprehensive handling of missing data with `fillna()` and `dropna()`.
- **NumPy:** The foundation for numerical computing in Python, providing efficient multi-dimensional array objects (ndarrays) and operations for linear algebra, random number generation, and element-wise mathematical functions. NumPy arrays consume less memory and execute faster than native Python lists.
- **Visualization Libraries:** Matplotlib provides low-level, highly customizable plotting (line charts, bar charts, histograms, scatter plots), while Seaborn builds on Matplotlib to offer statistical visualizations with cleaner defaults (heatmaps, pair plots, distribution plots).
- **scikit-learn:** The standard library for machine learning, offering a unified API for classification, regression, clustering, dimensionality reduction, and model evaluation, along with preprocessing utilities like scalers, encoders, and train/test splitting.
- **Jupyter Notebooks:** An interactive development environment that combines executable code, rich text (Markdown), and visualizations in a single document, making it ideal for exploratory data analysis, documentation, and sharing reproducible analytical workflows.
- **Supporting Libraries:** SciPy (advanced scientific and statistical computing), statsmodels (regression analysis, hypothesis testing, time-series analysis), and SQLAlchemy (database connectivity for integrating SQL queries into Python workflows).

**Practical Applications:** A typical Python analytics workflow involves reading data with `pandas.read_csv()`, exploring it with `.describe()` and Seaborn visualizations, cleaning and transforming with pandas operations, building a predictive model with scikit-learn, and presenting results in a Jupyter Notebook that serves as both analysis and documentation.

### SQL

SQL (Structured Query Language) is the standard programming language for managing and querying data in relational database systems. Developed in the 1970s, it remains the most widely used method of accessing and manipulating structured data and is an essential skill for data analysts working at any scale. SQL enables users to define database structures, insert and modify data, and write queries that retrieve precisely the information needed for analysis.

- **Command Categories:** DDL (Data Definition Language: CREATE, ALTER, DROP for defining tables and schemas), DML (Data Manipulation Language: INSERT, UPDATE, DELETE for modifying data), DQL (Data Query Language: SELECT for retrieving data), and DCL (Data Control Language: GRANT, REVOKE for managing permissions).
- **Core Query Operations:** Data retrieval with SELECT, filtering rows with WHERE clauses, sorting with ORDER BY, eliminating duplicates with DISTINCT, and limiting results with LIMIT/TOP. These form the building blocks of every SQL query.
- **Joins and Relationships:** Combining data across multiple tables using INNER JOIN (matching rows only), LEFT/RIGHT JOIN (preserving unmatched rows from one table), FULL OUTER JOIN, and CROSS JOIN. Understanding join logic is critical for working with normalized relational databases.
- **Aggregation and Grouping:** Functions like COUNT, SUM, AVG, MIN, and MAX summarize data, used with GROUP BY to compute metrics per category and HAVING to filter grouped results. These operations replace spreadsheet pivot tables at scale.
- **Window Functions:** Advanced analytical functions (ROW_NUMBER, RANK, DENSE_RANK, LAG, LEAD, running totals) that compute values across sets of rows related to the current row without collapsing the result set, enabling sophisticated row-level analytics.
- **Subqueries and CTEs:** Subqueries (nested SELECT statements) and Common Table Expressions (WITH clauses) allow complex queries to be broken into readable, modular components for multi-step data retrieval and transformation.

**Practical Applications:** An analyst might write a SQL query that joins a sales transactions table with a products table and a customers table, filters for the last quarter, groups by product category and region, and calculates total revenue and average order size -- extracting a complete business summary from millions of records in seconds.

### Data cleaning

Data cleaning (also called data cleansing or data scrubbing) is the process of identifying and correcting errors, inconsistencies, and anomalies in datasets to improve data quality before analysis. It involves screening for problems, diagnosing their causes, and applying appropriate corrective measures to ensure data is accurate, complete, and reliable. Research consistently shows that poor data quality leads to flawed analytical results, making systematic data cleaning essential before any modeling or reporting effort.

- **Handling Missing Values:** Common strategies include deletion (listwise or pairwise removal of incomplete records), imputation (replacing missing entries with the mean, median, mode, or model-predicted values), and flagging (adding indicator variables to preserve information about absent data). Advanced techniques such as multiple imputation create several plausible datasets and pool results for more robust inference.
- **Outlier Detection and Treatment:** Outliers are identified using statistical methods such as the interquartile range (IQR) method (values outside Q1 - 1.5*IQR to Q3 + 1.5*IQR), z-scores (values beyond 2-3 standard deviations), or visualization techniques like box plots and scatter plots. Treatment options include removal, capping/winsorizing, or transformation (e.g., log transformation).
- **Deduplication:** Identifying and removing duplicate records caused by data entry errors, system merges, or repeated imports, using exact matching or fuzzy matching algorithms for near-duplicate detection.
- **Standardization and Consistency:** Ensuring uniform formats for dates, phone numbers, addresses, and categorical labels across the dataset, correcting typos, and resolving conflicting entries from merged sources.
- **Validation Rules:** Applying domain-specific constraints (e.g., age must be positive, dates must be chronological) to flag records that violate logical or business rules.
- **Documentation:** Maintaining a record of all cleaning steps taken, values modified, and records removed to ensure reproducibility and transparency in the analytical pipeline.

**Practical Applications:** In a real-world analytics project, a data analyst might use Python's pandas library to detect null values with `isnull()`, identify outliers with IQR-based filtering, standardize inconsistent category labels using `replace()` or `map()`, and remove duplicate rows with `drop_duplicates()`, documenting each step in a Jupyter Notebook for reproducibility.

### ETL

ETL (Extract, Transform, Load) is a three-phase data integration process that combines, cleans, and organizes data from multiple sources into a single, consistent dataset for storage in a data warehouse, data lake, or other target system. It is the foundational mechanism for building reliable analytical data pipelines. ETL processes automate the movement of data across systems, ensuring that analysts and decision-makers have access to timely, accurate, and well-structured information.

- **Extract:** Raw data is copied or exported from heterogeneous source systems -- relational databases, flat files (CSV, JSON, XML), APIs, cloud services, and IoT streams -- into a staging area for processing. Extraction can be full (complete dataset) or incremental (only new/changed records).
- **Transform:** Data is cleansed, validated, deduplicated, and restructured to conform to a target schema. Common transformations include format standardization, data type conversions, aggregation, filtering, applying business rules, and joining data from multiple sources.
- **Load:** The transformed data is written to the target destination such as a data warehouse, data mart, data lake, or analytical platform. Loading strategies include full refresh (replacing all data) or incremental loading (appending or updating only changed records).
- **ELT Variant:** Extract, Load, Transform (ELT) reverses the last two steps by loading raw data into the target system first and then transforming it in-place, leveraging the processing power of modern cloud data warehouses like Snowflake, BigQuery, or Redshift.
- **Batch vs. Real-Time:** Traditional ETL runs on scheduled batch intervals (hourly, daily), while modern streaming ETL (using tools like Apache Kafka or Spark Streaming) processes data in near real-time for time-sensitive applications.
- **Common Tools:** Popular ETL tools include Apache Airflow (workflow orchestration), dbt (transformation layer), Informatica, Talend, AWS Glue, and Azure Data Factory, as well as Python-based custom pipelines using pandas and SQLAlchemy.

**Practical Applications:** A company might build an ETL pipeline that extracts daily sales data from a PostgreSQL transactional database, transforms it by joining with product and customer dimension tables and calculating revenue metrics, then loads the results into a Snowflake data warehouse where analysts can run dashboards and reports.
