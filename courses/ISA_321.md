### Advanced Java

**Definition:** Advanced Java encompasses intermediate and expert-level features of the Java programming language, building on core OOP principles such as inheritance, polymorphism, encapsulation, and abstraction to create robust, maintainable software systems.

- **Inheritance** allows one class to acquire fields and methods of another using the `extends` keyword, supporting single, multilevel, and hierarchical relationships
- **Polymorphism** enables objects to take "many forms" through method overriding (runtime) and method overloading (compile-time)
- **Encapsulation** wraps data and methods into a single class unit, restricting direct access via access modifiers
- **Abstraction** hides implementation details and exposes only essential features through abstract classes and interfaces
- **Advanced features** include lambda expressions, streams API, multithreading, and the Collections framework

**Practical relevance:** Mastering advanced Java is essential for building enterprise-grade applications, Android development, and backend services. These OOP principles form the foundation for design patterns and scalable software architecture used across the industry.

### Recursion

**Definition:** Recursion is a method of solving computational problems where a function calls itself to break a problem into smaller instances of the same problem, eventually reaching a base case that terminates the process.

- **Base case** defines the stopping condition that prevents infinite recursion and provides a direct result
- **Recursive case** is where the function calls itself with a modified argument, moving toward the base case
- **Single recursion** involves one self-reference (e.g., factorial, linear search), while **multiple recursion** involves two or more (e.g., Fibonacci, tree traversal)
- **Common applications** include merge sort, binary search, depth-first search, and directory traversal
- **Call stack** tracks each recursive call; excessive depth can cause a stack overflow error

**Practical relevance:** Recursion provides elegant solutions for problems with naturally recursive structures such as trees, graphs, and divide-and-conquer algorithms. Understanding recursion is fundamental to algorithm design and is a core topic in technical interviews and competitive programming.

### File I/O

**Definition:** Java File I/O (Input/Output) refers to the mechanisms in the `java.io` and `java.nio` packages for reading data from and writing data to files, streams, and other data sources.

- **Byte streams** (`InputStream`/`OutputStream`) handle raw binary data; key classes include `FileInputStream` and `FileOutputStream`
- **Character streams** (`Reader`/`Writer`) handle 16-bit Unicode text data with automatic encoding/decoding; key classes include `FileReader` and `FileWriter`
- **Buffered streams** (`BufferedReader`/`BufferedWriter`) wrap other streams to improve performance by reducing disk access through internal buffering
- **Standard streams** include `System.in` (keyboard input), `System.out` (console output), and `System.err` (error output)
- **NIO (New I/O)** provides channels, buffers, and selectors for high-performance, non-blocking I/O operations

**Practical relevance:** File I/O is essential for any application that persists data, processes log files, reads configuration, or exchanges data between systems. Understanding the difference between byte and character streams, and when to use buffering, is critical for writing efficient and correct Java programs.

### Exceptions

**Definition:** Java exceptions are events that disrupt the normal flow of a program. Java provides a structured exception handling mechanism using `try`, `catch`, `finally`, `throw`, and `throws` to manage errors gracefully at runtime.

- **Checked exceptions** (e.g., `IOException`, `SQLException`) are verified at compile-time and must be handled with try-catch or declared with `throws`
- **Unchecked exceptions** (e.g., `NullPointerException`, `ArrayIndexOutOfBoundsException`) extend `RuntimeException` and are not enforced at compile-time
- **Try-catch blocks** allow code to attempt risky operations and define recovery logic if an exception occurs
- **The `finally` block** executes regardless of whether an exception was thrown, commonly used for resource cleanup
- **Custom exceptions** can be created by extending `Exception` or `RuntimeException` to handle application-specific error conditions

**Practical relevance:** Robust exception handling is critical for building reliable software. It prevents application crashes, enables meaningful error messages for users, and ensures resources like database connections and file handles are properly released. Understanding checked vs. unchecked exceptions is a frequent topic in Java certification exams and interviews.

### Generics

**Definition:** Java Generics, introduced in J2SE 5.0, allow classes, interfaces, and methods to operate on parameterized types, enabling compile-time type safety and eliminating the need for explicit type casting.

- **Type parameters** (e.g., `<T>`, `<E>`, `<K, V>`) let you write code that works with different data types while maintaining type safety
- **Type safety** means the compiler catches type mismatches at compile-time rather than at runtime (e.g., adding an `Integer` to a `List<String>` produces a compile error)
- **Type erasure** removes generic type information at runtime, meaning generics are a compile-time-only feature on the JVM
- **Bounded type parameters** (`<T extends Number>`) restrict the types that can be used as arguments
- **Wildcards** (`?`, `? extends T`, `? super T`) provide flexibility when working with parameterized types in method parameters

**Practical relevance:** Generics are heavily used in the Java Collections Framework (`ArrayList<T>`, `HashMap<K,V>`) and throughout enterprise Java development. They reduce code duplication, improve readability, and catch bugs early. Understanding generics is essential for writing reusable libraries and APIs.

### Data structures

**Definition:** Data structures are specialized formats for organizing, storing, and managing data so it can be accessed and modified efficiently. Choosing the right data structure is fundamental to writing performant software.

- **Arrays** store elements of the same type in contiguous memory locations, providing O(1) access by index but fixed size
- **Linked lists** consist of nodes where each node holds data and a reference to the next node, enabling efficient insertion and deletion but O(n) access
- **Stacks** follow Last-In, First-Out (LIFO) ordering, used in undo operations, expression parsing, and function call management
- **Queues** follow First-In, First-Out (FIFO) ordering, used in scheduling, buffering, and breadth-first search
- **Trees** are hierarchical structures (e.g., binary trees, BSTs, AVL trees) used for searching, sorting, and hierarchical data representation
- **Hash tables** provide near O(1) average-case lookup using key-value pairs and hash functions

**Practical relevance:** Data structures form the backbone of efficient software. Selecting the appropriate structure for a given problem directly impacts performance, memory usage, and code clarity. They are a core subject in computer science education and technical interviews.

### Algorithm analysis

**Definition:** Algorithm analysis is the process of evaluating an algorithm's efficiency in terms of time complexity (how long it takes) and space complexity (how much memory it uses) as input size grows, primarily expressed using Big O notation.

- **Big O notation** describes the upper bound (worst-case) growth rate: O(1) constant, O(log n) logarithmic, O(n) linear, O(n log n) linearithmic, O(n^2) quadratic, O(2^n) exponential
- **Time complexity** measures the number of operations relative to input size; e.g., binary search is O(log n) vs. linear search at O(n)
- **Space complexity** measures memory consumption; e.g., an in-place sort uses O(1) extra space while merge sort uses O(n)
- **Best, average, and worst case** scenarios provide different perspectives on algorithm behavior
- **Practical comparison:** sorting 1 million items with bubble sort O(n^2) requires roughly 1 trillion operations, while merge sort O(n log n) needs only about 20 million

**Practical relevance:** Algorithm analysis guides developers in choosing optimal solutions for real-world problems. Understanding complexity classes is essential for writing scalable code, passing technical interviews, and making informed trade-offs between speed and memory in production systems.
