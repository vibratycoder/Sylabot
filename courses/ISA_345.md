### HTML

HTML (HyperText Markup Language) is the standard markup language used to structure content on the web. It uses a system of elements (tags) to wrap and annotate text, images, and other media so that web browsers can interpret and render them correctly. HTML forms the foundational layer of every web page, defining the document's structure and semantic meaning.

- **Elements and Tags**: HTML documents are built from elements such as `<head>`, `<body>`, `<p>`, `<div>`, `<span>`, `<img>`, `<ul>`, `<ol>`, and `<a>`. Each element defines a specific type of content or behavior on the page.
- **Semantic HTML**: Semantic elements like `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, and `<footer>` convey meaning about the content they contain, improving both readability and accessibility.
- **Document Structure**: Every HTML page follows a standard structure: a `<!DOCTYPE>` declaration, a root `<html>` element, a `<head>` section for metadata, and a `<body>` section for visible content.
- **Forms and Input**: HTML provides form elements (`<form>`, `<input>`, `<select>`, `<textarea>`) that enable user interaction and data collection on web pages.
- **Hyperlinks and Navigation**: The `<a>` (anchor) tag creates hyperlinks that connect pages within a site or across the web, forming the core navigational structure of the internet.
- **SEO Benefits**: Search engines give more importance to keywords inside semantic headings and links than those in non-semantic containers, making proper HTML structure critical for discoverability.

**Practical Applications:** Properly structured HTML is essential for building accessible, SEO-friendly websites. Developers use semantic HTML to ensure content is interpretable by browsers, screen readers, and search engine crawlers alike.

*Sources: [MDN Web Docs - HTML](https://developer.mozilla.org/en-US/docs/Web/HTML), [W3Schools - Semantic Elements](https://www.w3schools.com/html/html5_semantic_elements.asp), [web.dev - Semantic HTML](https://web.dev/learn/html/semantic-html)*

### JavaScript

JavaScript is a dynamic, high-level programming language that enables interactive and dynamic behavior on web pages. It runs natively in all modern browsers and allows developers to manipulate page content, respond to user actions, and communicate with servers without reloading the page. JavaScript is the programming layer of the client-side web stack, complementing HTML (structure) and CSS (presentation).

- **DOM Manipulation**: The Document Object Model (DOM) is a programming interface that represents the page as a tree of nodes. JavaScript can access and modify elements using methods like `getElementById()`, `querySelector()`, `createElement()`, and properties like `innerHTML` and `textContent`.
- **Event Handling**: JavaScript responds to user interactions (clicks, key presses, form submissions) through event listeners. The `addEventListener()` method is the most flexible approach, supporting multiple listeners per element and event propagation control (capturing and bubbling phases).
- **Variables and Data Types**: JavaScript supports variables declared with `let`, `const`, and `var`, along with data types including strings, numbers, booleans, arrays, objects, null, and undefined.
- **Functions and Scope**: Functions can be declared traditionally, as expressions, or as arrow functions. JavaScript uses lexical scoping and closures to manage variable accessibility.
- **Asynchronous Programming**: JavaScript handles asynchronous operations through callbacks, Promises, and the `async/await` syntax, enabling non-blocking operations like API calls and timers.
- **ES6+ Features**: Modern JavaScript includes template literals, destructuring, spread/rest operators, modules (`import`/`export`), and class syntax for cleaner, more expressive code.

**Practical Applications:** JavaScript powers everything from form validation and dynamic content updates to complex single-page applications, making it indispensable for modern web development.

*Sources: [MDN Web Docs - DOM Introduction](https://developer.mozilla.org/en-US/docs/Web/API/Document_Object_Model/Introduction), [MDN - Events](https://developer.mozilla.org/en-US/docs/Learn_web_development/Core/Scripting/Events), [W3Schools - JavaScript DOM](https://www.w3schools.com/js/js_htmldom.asp)*

### jQuery

jQuery is a fast, lightweight, and feature-rich JavaScript library designed to simplify HTML document traversal, DOM manipulation, event handling, animation, and AJAX interactions. Its motto, "write less, do more," reflects how it condenses many lines of vanilla JavaScript into single method calls. jQuery provides a consistent API that abstracts away browser compatibility differences, making cross-browser development significantly easier.

- **DOM Traversal and Manipulation**: jQuery uses CSS-style selectors via the `$()` function to select elements, then provides chaining methods like `.html()`, `.text()`, `.append()`, `.remove()`, `.addClass()`, and `.css()` to modify the DOM with minimal code.
- **Event Handling**: jQuery simplifies event binding with methods like `.on()`, `.click()`, `.hover()`, and `.submit()`. It normalizes event behavior across browsers and supports event delegation for dynamically added elements.
- **AJAX Requests**: The `$.ajax()` method and shorthand helpers (`$.get()`, `$.post()`, `$.getJSON()`) allow asynchronous server communication without page reloads. jQuery also provides global AJAX event handlers such as `ajaxStart`, `ajaxComplete`, and `ajaxError` for centralized request monitoring.
- **Animations and Effects**: Built-in methods like `.fadeIn()`, `.slideUp()`, `.animate()`, and `.toggle()` enable smooth visual transitions and UI effects without CSS animations.
- **Plugins and Extensibility**: jQuery has a vast ecosystem of plugins that extend its functionality for tasks like form validation, carousels, date pickers, and data tables.
- **Chaining**: jQuery methods return the jQuery object, allowing multiple operations to be chained together in a single statement for concise, readable code.

**Practical Applications:** jQuery remains widely used in legacy web applications and content management systems like WordPress. It is especially valuable for rapid prototyping and projects requiring broad browser compatibility without a full framework.

*Sources: [jQuery API Documentation](https://api.jquery.com/), [W3Schools - jQuery Introduction](https://www.w3schools.com/jquery/jquery_intro.asp), [jQuery AJAX Documentation](https://api.jquery.com/category/ajax/)*

### CSS

Cascading Style Sheets (CSS) is a stylesheet language used to describe the visual presentation of HTML documents, controlling layout, colors, fonts, spacing, and responsive behavior. CSS separates content (HTML) from presentation, allowing developers to maintain consistent styling across an entire website from centralized stylesheets. The "cascade" refers to the set of rules that determine which styles take precedence when multiple rules apply to the same element.

- **Selectors and Specificity**: CSS targets elements using selectors (element, class, ID, attribute, pseudo-class, pseudo-element). Specificity determines which rule wins when conflicts arise, following the hierarchy: inline styles > IDs > classes > elements.
- **Box Model**: Every element is rendered as a rectangular box composed of content, padding, border, and margin. Understanding the box model is fundamental to controlling element sizing and spacing.
- **Flexbox**: A one-dimensional layout system activated with `display: flex` that distributes space among items along a single axis (row or column). Properties like `justify-content`, `align-items`, and `flex-wrap` control alignment and wrapping behavior.
- **CSS Grid**: A two-dimensional layout system that manages rows and columns simultaneously using `display: grid`. Grid enables complex page layouts with properties like `grid-template-columns`, `grid-template-rows`, and `grid-area`.
- **Responsive Design**: Media queries (`@media`) allow styles to adapt based on viewport size, device type, or orientation. Combined with relative units (`em`, `rem`, `%`, `vw`, `vh`), they enable layouts that work across desktops, tablets, and phones.
- **Transitions and Animations**: CSS provides `transition` properties for smooth state changes and `@keyframes` animations for more complex, multi-step visual effects without JavaScript.

**Practical Applications:** CSS is essential for creating visually appealing, responsive web interfaces. Modern layout techniques like Flexbox and Grid have replaced older float-based methods, enabling developers to build complex layouts with cleaner, more maintainable code.

*Sources: [MDN Web Docs - CSS](https://developer.mozilla.org/en-US/docs/Web/CSS), [MDN - Flexbox Basics](https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Flexible_box_layout/Basic_concepts), [W3Schools - CSS Grid](https://www.w3schools.com/css/css_grid.asp)*

### Web design principles

Web design principles are the foundational guidelines that govern how digital interfaces are structured to be visually effective, usable, and engaging. These principles draw from graphic design, cognitive psychology, and user experience research to ensure that websites communicate clearly and guide users toward desired actions. A well-designed interface reduces cognitive load, establishes trust, and improves overall user satisfaction.

- **Visual Hierarchy**: The arrangement of design elements by importance using size, color, contrast, and placement to guide the user's eye through the page in a deliberate order. Larger, bolder, and higher-contrast elements naturally draw attention first.
- **Typography**: Font selection, sizing, and spacing convey tone and improve readability. Best practices include limiting typefaces to 2-3 per page, using sans-serif fonts (such as Arial, Helvetica, or Roboto) for screen legibility, and establishing clear heading hierarchies.
- **Color Theory**: Strategic use of color evokes emotions and reinforces brand identity. The 60-30-10 rule (60% dominant hue, 30% secondary, 10% accent) creates visual balance and smooth transitions between focal points.
- **Whitespace (Negative Space)**: Intentional empty space around elements reduces clutter, improves comprehension, and creates a sense of elegance. Whitespace is an active design element, not wasted space.
- **Consistency and Alignment**: Uniform styling of buttons, links, headings, and layouts builds familiarity and trust. Grid-based alignment creates order and makes interfaces feel professional.
- **User-Centered Design**: Effective web design prioritizes the needs, goals, and behaviors of the target audience. Usability testing and iterative feedback loops ensure the interface serves its users well.

**Practical Applications:** Applying these principles helps designers create interfaces that are not only visually appealing but also functional and conversion-optimized, whether for e-commerce checkouts, portfolio sites, or corporate dashboards.

*Sources: [NNGroup - Principles of Visual Design](https://www.nngroup.com/articles/principles-visual-design/), [IxDF - Visual Hierarchy](https://ixdf.org/literature/topics/visual-hierarchy), [Webflow - Visual Hierarchy](https://webflow.com/blog/visual-hierarchy)*

### Accessibility

Web accessibility ensures that websites and web applications are usable by all people, including those with disabilities such as visual, auditory, motor, or cognitive impairments. The Web Content Accessibility Guidelines (WCAG), published by the W3C, provide the internationally recognized standard for accessible web content. Accessibility is both a legal requirement in many jurisdictions and an ethical commitment to inclusive design.

- **WCAG Four Principles (POUR)**: All accessible content must be Perceivable (users can sense it), Operable (users can interact with it), Understandable (users can comprehend it), and Robust (content works across diverse technologies and assistive devices).
- **ARIA (Accessible Rich Internet Applications)**: WAI-ARIA provides a set of roles, states, and properties (e.g., `aria-label`, `aria-live`, `aria-expanded`) that supplement HTML to convey widget behavior and dynamic content updates to assistive technologies like screen readers.
- **Semantic HTML First**: The first rule of ARIA is to use native HTML elements with built-in semantics and behaviors whenever possible, rather than re-purposing generic elements with ARIA attributes. A `<button>` is inherently accessible; a `<div>` styled as a button is not.
- **Keyboard Navigation**: All interactive elements must be operable via keyboard alone. Proper focus management, visible focus indicators, and logical tab order are essential for users who cannot use a mouse.
- **Alternative Text and Captions**: Images require descriptive `alt` attributes, videos need captions and transcripts, and audio content should have text alternatives to serve users with sensory disabilities.
- **Color and Contrast**: Text must meet minimum contrast ratios (4.5:1 for normal text, 3:1 for large text per WCAG AA) and information should never be conveyed by color alone.

**Practical Applications:** Accessible websites reach a broader audience, comply with regulations like the ADA and Section 508, and often improve usability for all users. Accessibility auditing tools such as Lighthouse, axe, and WAVE help developers identify and fix issues.

*Sources: [W3C - WAI-ARIA Overview](https://www.w3.org/WAI/standards-guidelines/aria/), [MDN - ARIA](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA), [W3C - WCAG 2.1](https://www.w3.org/TR/WCAG21/)*

### Client-side development

Client-side development refers to all code and processes that execute in the user's web browser rather than on a remote server. It encompasses the three core web technologies -- HTML, CSS, and JavaScript -- along with libraries, frameworks, and tooling that run in the browser environment. Client-side development is responsible for everything the user sees and interacts with, from page layout and styling to dynamic content updates and form validation.

- **Browser Rendering Pipeline**: When a browser receives HTML, it parses the document into a DOM tree, applies CSS to build a render tree, calculates layout (reflow), and paints pixels to the screen. Understanding this pipeline is key to writing performant client-side code.
- **Single-Page Applications (SPAs)**: SPAs load a single HTML shell and use JavaScript to dynamically update the DOM as users navigate, avoiding full page reloads. Frameworks like React, Vue, and Angular facilitate this pattern by managing state and routing on the client.
- **Client-Side vs. Server-Side Rendering**: Client-side rendering (CSR) builds the page in the browser using JavaScript, while server-side rendering (SSR) generates HTML on the server before sending it to the client. Hybrid approaches combine both for optimal performance and SEO.
- **Development Tools**: Modern browsers include developer tools (DevTools) with features for inspecting the DOM, debugging JavaScript, monitoring network requests, profiling performance, and auditing accessibility.
- **Build Tools and Module Bundlers**: Tools like Webpack, Vite, and Parcel compile, bundle, and optimize client-side assets (JavaScript modules, CSS preprocessors, images) for production deployment.
- **State Management**: As applications grow in complexity, managing data across components becomes critical. Patterns like component state, context/provider, and external stores (Redux, Pinia) help organize and synchronize application data on the client.

**Practical Applications:** Client-side development skills are essential for building responsive, interactive web applications. Understanding the trade-offs between CSR and SSR helps developers choose the right architecture for performance, SEO, and user experience requirements.
