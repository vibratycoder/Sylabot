### Structured programming

Structured programming is a programming paradigm that organizes code using clear, logical control flow constructs rather than arbitrary jumps (such as `goto` statements). Introduced in the late 1960s and popularized by Edsger Dijkstra's landmark paper "Go To Statement Considered Harmful," it emphasizes writing programs composed exclusively of three fundamental control structures: sequence, selection, and iteration. The approach dramatically improves code readability, maintainability, and correctness.

- **Sequence**: Statements are executed one after another in a defined, linear order from top to bottom.
- **Selection**: Conditional branching (e.g., `if/else`, `switch`) allows the program to choose different execution paths based on a condition's truth value.
- **Iteration**: Loop constructs (`for`, `while`, `do-while`) enable repeated execution of a code block until a termination condition is met.
- **Elimination of goto**: The structured program theorem (Bohm-Jacopini, 1966) proved that any computable function can be expressed using only sequence, selection, and iteration -- making `goto` unnecessary and reducing "spaghetti code."
- **Modular decomposition**: Programs are broken into self-contained subroutines or modules, each responsible for a specific task, promoting code reuse and easier debugging.
- **Block structure**: Related statements are grouped into blocks (delimited by braces or indentation), clarifying scope and logical grouping.

**Practical Applications:** Structured programming is the foundation of virtually all modern software development. Whether writing a simple script or a large enterprise system, developers rely on its principles to produce code that is easy to read, test, and maintain.

### Control structures

Control structures are programming constructs that govern the flow of execution within a program, determining which statements run, in what order, and how many times. Without control structures, a program would simply execute every line from top to bottom with no ability to make decisions or repeat actions. They are the building blocks that give programs their logic and dynamic behavior.

- **Sequential control**: The default mode of execution where statements run one after another in the order they appear. Every program relies on sequential flow as its baseline.
- **Selection (conditional) control**: Allows the program to choose between different code paths based on Boolean conditions. Common forms include `if`, `if-else`, `else if` chains, and `switch/case` statements.
- **Iteration (loop) control**: Enables a block of code to execute repeatedly. `for` loops run a predetermined number of times; `while` loops continue as long as a condition remains true; `do-while` loops guarantee at least one execution before checking the condition.
- **Nested control structures**: Control structures can be placed inside one another (e.g., a loop inside an `if` block, or an `if` inside a loop), enabling complex decision-making and data processing patterns.
- **Break and continue**: Auxiliary control keywords that modify loop behavior -- `break` exits a loop early, while `continue` skips the current iteration and proceeds to the next.
- **Boolean expressions**: The conditions that drive selection and iteration are evaluated as `true` or `false` using comparison operators (`==`, `!=`, `<`, `>`) and logical operators (`&&`, `||`, `!`).

**Practical Applications:** Control structures are essential in every real-world program, from validating user input with `if` statements to processing large datasets with `for` loops, to building menu-driven applications with `switch/case` and `while` loops.

### Object-oriented programming

Object-oriented programming (OOP) is a programming paradigm that organizes software design around objects -- self-contained entities that bundle data (attributes) and behavior (methods) together. Rather than thinking of a program as a sequence of instructions, OOP models it as a collection of interacting objects, each responsible for its own state and operations. OOP is the dominant paradigm in modern languages such as Java, Python, C++, and C#.

- **Encapsulation**: Data and the methods that operate on it are wrapped together inside a class, with access controlled through visibility modifiers (`private`, `public`, `protected`). This hides internal implementation details and exposes only a controlled interface via getter/setter methods.
- **Inheritance**: A class (subclass) can inherit attributes and methods from another class (superclass), establishing an "is-a" relationship. This promotes code reuse and creates a natural hierarchy (e.g., `Dog` inherits from `Animal`).
- **Polymorphism**: Objects of different classes can respond to the same method call in their own way. Compile-time polymorphism is achieved through method overloading (same method name, different parameters), while runtime polymorphism uses method overriding (subclass provides its own implementation of a parent method).
- **Abstraction**: Complex implementation details are hidden behind simple interfaces. Abstract classes and interfaces define what an object can do without specifying how it does it, allowing developers to work at a higher conceptual level.
- **Message passing**: Objects communicate by sending messages (calling methods) on one another, promoting loose coupling and modular design.
- **Composition vs. inheritance**: Modern OOP also emphasizes "has-a" relationships (composition), where objects contain references to other objects, as a flexible alternative to deep inheritance hierarchies.

**Practical Applications:** OOP powers most large-scale software systems, from mobile apps and web frameworks to game engines and enterprise platforms, because its modular structure makes complex codebases easier to develop, extend, and maintain collaboratively.

### Classes

A class is a blueprint or template in object-oriented programming that defines the structure and behavior of objects. It specifies what data an object will hold (attributes/fields) and what operations it can perform (methods/functions). When you create an object from a class, that object is called an instance, and it receives its own copy of the class's attributes while sharing the method definitions.

- **Attributes (fields/properties)**: Variables declared within a class that store the state of each object. Instance attributes are unique to each object (e.g., a `Student` object's `name` and `gpa`), while class attributes are shared across all instances.
- **Methods**: Functions defined inside a class that describe the behaviors an object can perform. Common categories include accessor methods (getters), mutator methods (setters), and utility/helper methods.
- **Constructors**: Special methods (e.g., `__init__` in Python, or a same-named method in Java) that are automatically called when a new object is instantiated. They initialize the object's attributes to starting values.
- **Instantiation**: The process of creating a concrete object from a class template using the `new` keyword (Java/C++) or by calling the class directly (Python). Each instance occupies its own space in memory.
- **Access modifiers**: Keywords like `public`, `private`, and `protected` that control which parts of a class are visible to outside code, enforcing encapsulation.
- **The `this`/`self` reference**: A keyword used inside a class to refer to the current instance, allowing methods to access and modify the object's own attributes and call its own methods.

**Practical Applications:** Classes are the fundamental organizational unit in OOP-based software. Developers use them to model real-world entities -- such as bank accounts, users, or products -- turning complex systems into manageable, reusable components.

### Debugging

Debugging is the systematic process of identifying, isolating, and fixing errors (bugs) in a computer program. It is one of the most critical skills a programmer can develop, as even experienced developers spend a significant portion of their time diagnosing and resolving defects. Modern Integrated Development Environments (IDEs) such as Visual Studio, IntelliJ IDEA, and PyCharm provide powerful built-in debugging tools that make this process far more efficient than manual inspection alone.

- **Breakpoints**: Markers placed on specific lines of code that instruct the debugger to pause execution at that point. When paused, developers can inspect variable values, the call stack, and memory state. Conditional breakpoints pause only when a specified condition is true, which is invaluable for tracking intermittent bugs.
- **Stepping through code**: Once paused at a breakpoint, developers use "Step Over" (execute the current line and move to the next), "Step Into" (enter a called function to trace its execution), and "Step Out" (finish the current function and return to the caller) to follow execution line by line.
- **Watch variables and expressions**: Debuggers allow you to monitor specific variables or expressions in real time, observing how their values change as the program executes.
- **Print/log debugging**: Inserting print statements or log messages at strategic points to trace program flow and variable values -- a simple but effective technique, especially in environments without a graphical debugger.
- **Binary search debugging**: Placing breakpoints or print statements at the midpoint of suspect code, then narrowing the search area by half each time until the bug is located.
- **Rubber duck debugging**: Explaining your code line by line to another person (or an inanimate object) to force yourself to articulate assumptions and often discover the error in the process.

**Practical Applications:** Debugging is an everyday activity in professional software development. From fixing a crashing mobile app to resolving a data corruption issue in a database-driven system, effective debugging skills directly determine how quickly a team can ship reliable software.

### Program design

Program design is the planning phase of software development in which a programmer maps out the logic, structure, and flow of a program before writing any actual code. Good design reduces errors, simplifies implementation, and produces software that is easier to test and maintain. It bridges the gap between understanding a problem and implementing a working solution.

- **Top-down design (stepwise refinement)**: A strategy that starts with the overall problem and progressively breaks it into smaller, more manageable sub-problems. Each sub-problem is refined until it is simple enough to implement directly as a function or module.
- **Pseudocode**: An informal, language-independent way of expressing an algorithm using a mixture of natural language and programming-like syntax. It allows developers to focus on logic without worrying about syntax rules, making it easier to spot logical errors early.
- **Flowcharts**: Visual diagrams that represent program logic using standardized symbols -- ovals for start/end, rectangles for processes, diamonds for decisions, and arrows for flow direction. They provide an intuitive, at-a-glance understanding of an algorithm's structure.
- **Modular design**: Dividing a program into independent, self-contained modules (functions or classes), each responsible for a single task. This promotes code reuse, parallel development, and easier testing.
- **Algorithm development**: Designing a step-by-step procedure to solve a specific problem, considering efficiency (time and space complexity) and correctness before coding begins.
- **Input-Process-Output (IPO) model**: A foundational design framework that defines what data a program receives (input), what operations it performs (process), and what results it produces (output).

**Practical Applications:** Program design techniques are used in every phase of professional software engineering, from whiteboard sessions during sprint planning to formal system architecture documents. Teams that invest in design before coding consistently deliver higher-quality software with fewer defects.
