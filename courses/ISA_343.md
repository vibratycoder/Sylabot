### Cloud computing

Cloud computing is the delivery of computing services -- including servers, storage, databases, networking, software, and analytics -- over the internet ("the cloud") to offer flexible resources, rapid innovation, and economies of scale. It allows organizations to avoid the upfront cost and complexity of owning and maintaining their own IT infrastructure, instead paying only for what they use on a consumption basis. The three primary service models -- IaaS, PaaS, and SaaS -- represent different levels of abstraction and management responsibility.

- **Infrastructure as a Service (IaaS):** Provides on-demand virtualized computing resources (compute, storage, networking) over the internet. Customers manage the OS, middleware, and applications while the provider manages the physical hardware. Examples include AWS EC2, Microsoft Azure VMs, and Google Compute Engine.
- **Platform as a Service (PaaS):** Delivers a managed environment for developing, testing, and deploying applications without managing underlying infrastructure. Developers focus on code while the provider handles OS patching, load balancing, and capacity planning. Examples include Heroku, Google App Engine, and Azure App Service.
- **Software as a Service (SaaS):** Provides fully managed, ready-to-use applications accessible via web browser. The provider handles everything from infrastructure to application updates and security. Examples include Salesforce, Microsoft 365, and Google Workspace.
- **Deployment Models:** Public cloud (shared infrastructure), private cloud (dedicated to one organization), hybrid cloud (combination of public and private), and multi-cloud (services from multiple providers).
- **Key Benefits:** Elasticity and scalability, pay-as-you-go pricing, global availability, disaster recovery, and reduced time to market for new services.
- **Shared Responsibility Model:** Security and management responsibilities are divided between the cloud provider and the customer, with the split varying by service model (IaaS gives the customer more control and responsibility; SaaS gives the provider more).

**Practical Applications:** Organizations use cloud computing to host web applications, run data analytics pipelines, store and back up enterprise data, and deploy machine learning models -- all without maintaining physical data centers.

### Operating systems

An operating system (OS) is system software that manages computer hardware and software resources and provides common services for application programs. It acts as an intermediary between users and the computer hardware, handling tasks such as memory allocation, process scheduling, file management, and device control. Major operating systems include Windows, macOS, Linux, and Unix, each with distinct architectures suited to different use cases from personal computing to enterprise servers.

- **Kernel:** The core component of the OS that operates at the lowest level, managing direct communication between hardware and software. It runs in a protected memory space (kernel space) and handles interrupt processing, memory management, and device drivers. Types include monolithic kernels (Linux), microkernels (Minix), and hybrid kernels (Windows NT).
- **Process Management:** The OS creates, schedules, and terminates processes. Each process receives a unique Process ID (PID) and a Process Control Block (PCB) that tracks its state. Scheduling algorithms (round-robin, priority-based, shortest job first) determine how CPU time is allocated among competing processes.
- **Memory Management:** The OS allocates and deallocates memory for processes, implements virtual memory through paging and segmentation, and prevents processes from accessing each other's memory space. This ensures both efficient resource use and system stability.
- **File Systems:** The OS organizes data on storage devices using file systems (NTFS, ext4, APFS) that manage file creation, deletion, reading, writing, and access permissions through a hierarchical directory structure.
- **I/O Management:** Device drivers and I/O schedulers coordinate communication between the CPU and peripheral devices (disks, keyboards, network adapters), abstracting hardware complexity from application software.
- **Security and Access Control:** The OS enforces user authentication, file permissions, and process isolation to protect system integrity and data confidentiality.

**Practical Applications:** Operating systems underpin every computing environment -- from smartphones (Android, iOS) to cloud data centers (Linux) -- making them essential knowledge for IT professionals managing infrastructure and deploying applications.

### Computer networks

A computer network is a collection of interconnected devices (computers, servers, routers, switches) that communicate and share resources using defined protocols and transmission media. Networks enable data exchange across local and global scales, forming the backbone of modern communication, commerce, and cloud services. Understanding network architecture requires familiarity with layered reference models, protocols, and physical/logical topologies.

- **OSI Model (7 Layers):** The Open Systems Interconnection model provides a theoretical framework with seven layers -- Physical, Data Link, Network, Transport, Session, Presentation, and Application. Each layer has specific responsibilities, from transmitting raw bits (Physical) to providing user-facing services (Application). The OSI model is primarily used for education and troubleshooting.
- **TCP/IP Model (5 Layers):** The practical model that powers the internet, consisting of Physical, Data Link, Network, Transport, and Application layers. TCP/IP is protocol-driven and maps directly to real-world implementations. Key protocols include IP (addressing and routing), TCP (reliable delivery), UDP (fast, connectionless delivery), HTTP/HTTPS (web traffic), and DNS (name resolution).
- **Network Topologies:** Physical and logical arrangements of network nodes including bus, star, ring, mesh, and hybrid configurations. Topology affects performance, fault tolerance, and scalability.
- **Network Devices:** Routers direct traffic between networks using IP addresses; switches forward frames within a local network using MAC addresses; firewalls filter traffic based on security rules; load balancers distribute workloads across servers.
- **IP Addressing and Subnetting:** IPv4 (32-bit) and IPv6 (128-bit) addressing schemes identify devices on a network. Subnetting divides networks into smaller segments for improved performance and security. CIDR notation and subnet masks define network boundaries.
- **Wireless and WAN Technologies:** Wi-Fi (802.11 standards), cellular networks (4G/5G), VPNs, and MPLS extend connectivity beyond local wired networks, enabling remote access and global communication.

**Practical Applications:** Network knowledge is essential for designing enterprise LANs, configuring cloud virtual networks (VPCs), troubleshooting connectivity issues, and ensuring secure data transmission across distributed systems.

### Middleware

Middleware is software that acts as a bridge between an operating system or database and applications, especially across a distributed network. Often described as "software glue," middleware provides services beyond what the OS offers -- such as message routing, authentication, API management, and data transformation -- enabling disparate systems and applications to communicate seamlessly. It abstracts the complexity of underlying infrastructure so developers can focus on business logic rather than low-level integration plumbing.

- **Message-Oriented Middleware (MOM):** Enables asynchronous communication between distributed systems using message queues and publish-subscribe patterns. Examples include Apache Kafka, RabbitMQ, and IBM MQ. MOM decouples senders and receivers, improving system resilience and scalability.
- **Remote Procedure Call (RPC) Middleware:** Allows a program to execute a procedure on a remote server as if it were a local call. gRPC and XML-RPC are common implementations that simplify distributed computing by hiding network communication details from developers.
- **Object Request Brokers (ORB):** Middleware based on object-oriented principles that enables objects in one application to invoke methods on objects in another application across a network. CORBA (Common Object Request Broker Architecture) is a well-known standard in this category.
- **Database Middleware:** Provides standardized access to databases regardless of the underlying DBMS. ODBC (Open Database Connectivity) and JDBC (Java Database Connectivity) allow applications to query different database systems through a unified interface.
- **API Gateways and Integration Platforms:** Modern middleware includes API gateways (Kong, Apigee) that manage API traffic, enforce security policies, and handle rate limiting. Enterprise integration platforms (MuleSoft, Dell Boomi) orchestrate complex workflows across SaaS applications, on-premises systems, and cloud services.
- **Web Middleware:** Application servers (Apache Tomcat, IIS, Nginx) and web frameworks that handle HTTP requests, session management, and serve dynamic content between web clients and back-end databases.

**Practical Applications:** Middleware is critical in enterprise environments where legacy systems must integrate with modern cloud applications, enabling organizations to build service-oriented architectures (SOA) and microservices without replacing existing infrastructure.

### Virtualization

Virtualization is technology that creates abstracted, simulated versions of computing resources -- including servers, storage, networks, and operating systems -- from a single physical hardware system. By decoupling software from hardware, virtualization enables multiple workloads to share the same physical infrastructure while remaining isolated from one another. It is the foundational technology that makes cloud computing economically viable and operationally flexible.

- **Hypervisors (Type 1 and Type 2):** A hypervisor is software that creates and manages virtual machines by sitting between hardware and guest operating systems. Type 1 (bare-metal) hypervisors run directly on hardware for maximum performance and are used in data centers (VMware ESXi, Microsoft Hyper-V, KVM). Type 2 (hosted) hypervisors run atop a conventional OS and are common for development and testing (VirtualBox, VMware Workstation).
- **Virtual Machines (VMs):** A VM is a software-defined computer with its own OS, CPU allocation, memory, and storage, running as an isolated guest on a physical host. Multiple VMs can coexist on one server, each running a different operating system and application stack.
- **Containers:** A lighter-weight alternative to VMs, containers virtualize at the OS level rather than the hardware level. Docker containers share the host OS kernel, making them faster to start and more resource-efficient. Kubernetes orchestrates containers at scale across clusters.
- **Network Virtualization:** Software-defined networking (SDN) and network function virtualization (NFV) abstract network resources, allowing administrators to configure and manage virtual networks, switches, and firewalls through software rather than physical hardware.
- **Storage Virtualization:** Pools physical storage from multiple devices into a single logical unit managed centrally. Technologies like SANs (Storage Area Networks) and software-defined storage simplify provisioning and improve utilization.
- **Desktop Virtualization (VDI):** Virtual Desktop Infrastructure delivers desktop environments from a centralized server, enabling users to access their workspace from any device. This improves security, simplifies management, and supports remote work.

**Practical Applications:** Enterprises use virtualization to consolidate servers (reducing hardware costs by 50-70%), enable rapid disaster recovery, create isolated development and testing environments, and build the elastic infrastructure that powers public and private clouds.

### Big Data infrastructure

Big Data infrastructure refers to the hardware, software, and frameworks required to capture, store, process, and analyze datasets that are too large, fast, or complex for traditional database systems. Characterized by the "3 Vs" -- Volume (scale of data), Velocity (speed of data flow), and Variety (diversity of data types) -- Big Data demands distributed architectures that can scale horizontally across clusters of commodity hardware. The Apache Hadoop ecosystem remains the foundational platform, though modern architectures increasingly incorporate real-time processing and cloud-native solutions.

- **Hadoop Distributed File System (HDFS):** The primary storage layer of Hadoop, HDFS splits large files into blocks (typically 128 MB) and replicates them across multiple nodes for fault tolerance. A NameNode manages metadata and file operations while DataNodes store the actual data blocks and handle read/write requests.
- **MapReduce:** Hadoop's original batch processing paradigm that divides computation into two phases -- Map (filters and sorts data) and Reduce (aggregates results). While powerful for batch workloads, MapReduce has been largely supplemented by faster frameworks like Apache Spark.
- **YARN (Yet Another Resource Negotiator):** The cluster resource management layer that schedules and allocates CPU, memory, and storage across applications running in the Hadoop ecosystem. A central ResourceManager coordinates with per-node NodeManagers to optimize workload distribution.
- **Apache Spark:** A fast, in-memory processing engine that supports batch processing, real-time streaming, machine learning (MLlib), and graph computation (GraphX). Spark can run 10-100x faster than MapReduce for iterative workloads due to its in-memory computation model.
- **Data Warehousing and Query Tools:** Apache Hive provides SQL-like querying over HDFS data; Apache Pig offers a high-level scripting language for data transformation; Presto and Apache Impala enable interactive, low-latency queries on massive datasets.
- **NoSQL Databases:** Non-relational databases designed for Big Data include HBase (column-family store on HDFS), Cassandra (distributed wide-column store), and MongoDB (document store). These systems prioritize scalability and availability over strict ACID compliance.

**Practical Applications:** Retailers use Big Data infrastructure to analyze customer purchasing patterns across millions of transactions in real time, while healthcare organizations process genomic datasets and electronic health records to support precision medicine and predictive analytics.

### Hardware

Computer hardware encompasses the physical components that make up a computing system, from processors and memory to storage devices and peripherals. Understanding hardware architecture is essential for IT professionals because infrastructure decisions -- such as choosing server configurations, storage solutions, and network equipment -- directly impact system performance, reliability, and cost. Modern hardware design follows the von Neumann architecture, where a central processor fetches and executes instructions from memory in a sequential cycle.

- **Central Processing Unit (CPU):** The "brain" of the computer, responsible for executing instructions and performing calculations. The CPU contains the Arithmetic Logic Unit (ALU) for mathematical and logical operations, the Control Unit (CU) for directing data flow, and registers for temporary high-speed storage. Key performance factors include clock speed (GHz), core count, cache size (L1/L2/L3), and instruction set architecture (x86, ARM).
- **Memory Hierarchy:** Computer memory is organized in tiers balancing speed, capacity, and cost. CPU registers and cache (SRAM) are fastest but smallest; RAM (DRAM) provides volatile working memory in gigabytes; storage (SSD/HDD) offers persistent capacity in terabytes. This hierarchy ensures the CPU accesses frequently-used data quickly while bulk data remains available at lower cost.
- **Storage Devices:** HDDs use spinning magnetic platters and are cost-effective for bulk storage. SSDs use NAND flash memory with no moving parts, delivering significantly faster read/write speeds, lower latency, and greater durability. NVMe SSDs connect directly via PCIe for even higher throughput.
- **Motherboard and Bus Architecture:** The motherboard connects all components through buses (PCIe, SATA, USB) and chipsets that manage data flow between CPU, memory, storage, and peripherals. Form factors (ATX, Micro-ATX, rack-mount) determine physical compatibility and expansion options.
- **Server Hardware:** Enterprise servers feature redundant power supplies, ECC (Error-Correcting Code) memory, hot-swappable drives, and multi-socket CPU configurations for high availability. Blade servers and rack-mount units are standard in data centers.
- **Networking Hardware:** Network Interface Cards (NICs), switches, routers, and fiber-optic transceivers provide connectivity. Modern data centers use 10/25/100 Gbps Ethernet and InfiniBand for high-speed interconnects.

**Practical Applications:** IT professionals must evaluate hardware specifications when provisioning servers for cloud infrastructure, designing high-performance computing clusters, or selecting storage solutions that balance cost with IOPS requirements for database workloads.
