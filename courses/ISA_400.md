### Neural networks

**Definition:** A neural network is a computational model inspired by the biological brain, consisting of interconnected units called artificial neurons organized into layers. Each neuron receives weighted inputs, sums them with a bias, and passes the result through an activation function to produce an output.

- **Architecture** consists of three layer types: input layer (receives external data), hidden layers (where learning occurs), and output layer (produces predictions)
- **Weights and biases** are learnable parameters adjusted during training to minimize prediction error
- **Activation functions** introduce non-linearity, enabling the network to learn complex patterns; common types include ReLU, sigmoid, tanh, and softmax
- **Without activation functions**, the network would only perform linear transformations regardless of depth, reducing it to a simple linear regression model
- **Training** involves forward propagation (computing outputs), loss calculation, and backpropagation (adjusting weights via gradient descent)

**Practical relevance:** Neural networks are the foundation of modern AI applications including image recognition, speech processing, natural language understanding, and recommendation systems. Understanding neurons, layers, and activation functions is prerequisite knowledge for all deep learning architectures.

### Deep learning

**Definition:** Deep learning is a subset of machine learning that uses neural networks with multiple hidden layers (hence "deep") to learn hierarchical representations of data. It powers breakthroughs in image recognition, language translation, generative AI, and autonomous systems.

- **Deep architectures** stack many layers where early layers detect simple features (edges, tones) and deeper layers compose them into complex patterns (faces, objects, sentences)
- **Backpropagation** is the key training algorithm that propagates errors backward through the network using the chain rule of calculus to compute gradients for each weight
- **Loss functions** (e.g., cross-entropy, mean squared error) measure the difference between predicted and actual values, guiding the optimization process
- **Gradient descent** iteratively updates weights in the direction that minimizes the loss; variants include SGD, Adam, and RMSprop
- **Regularization techniques** (dropout, batch normalization, weight decay) prevent overfitting and improve generalization to unseen data

**Practical relevance:** Deep learning has become the dominant approach for tasks involving unstructured data such as images, text, and audio. Frameworks like PyTorch and TensorFlow have made deep learning accessible to practitioners, driving adoption across healthcare, finance, robotics, and creative industries.

### PyTorch

**Definition:** PyTorch is an open-source deep learning framework developed by Meta AI that provides tensors with GPU acceleration and an automatic differentiation engine (Autograd) for building and training neural networks with a dynamic computational graph.

- **Tensors** are multi-dimensional arrays (similar to NumPy arrays) that serve as the fundamental data structure, supporting operations on both CPU and GPU for accelerated computation
- **Autograd** automatically computes gradients for all tensor operations using reverse-mode auto-differentiation, eliminating manual calculus for backpropagation
- **Dynamic computational graphs** are built on-the-fly during execution, allowing flexible model architectures and easy debugging with standard Python tools
- **Key modules** include `torch.nn` (neural network layers), `torch.optim` (optimizers like Adam and SGD), and `torch.utils.data` (dataset loading and batching)
- **Ecosystem** includes TorchVision (computer vision), TorchText (NLP), TorchAudio, and integration with Hugging Face Transformers

**Practical relevance:** PyTorch is the most widely used framework in deep learning research and is rapidly growing in industry adoption. Its Pythonic interface and dynamic graphs make it ideal for experimentation and prototyping, and it is the framework used in ISA 400 for implementing deep learning models.

### Natural language processing

**Definition:** Natural Language Processing (NLP) is a subfield of artificial intelligence focused on enabling computers to understand, interpret, generate, and respond to human language. It bridges the gap between human communication and machine computation.

- **Tokenization** is a fundamental preprocessing step that divides text into smaller units (tokens) such as words, subwords, or characters; subword tokenization is standard in transformer models like BERT and GPT
- **Key NLP tasks** include sentiment analysis, named entity recognition (NER), text classification, machine translation, question answering, and text summarization
- **Word embeddings** (Word2Vec, GloVe) and contextual embeddings (BERT, GPT) convert words into dense numerical vectors that capture semantic meaning
- **Transformer architecture** uses self-attention mechanisms to process entire sequences in parallel, replacing older sequential approaches and enabling models like GPT and BERT
- **Popular tools** include Hugging Face Transformers, spaCy, NLTK, and SentencePiece for various NLP tasks

**Practical relevance:** NLP powers chatbots, search engines, virtual assistants, translation services, and content moderation systems used by billions daily. In deep learning, NLP represents one of the most active and impactful application domains, with large language models (LLMs) transforming how humans interact with technology.

### Computer vision

**Definition:** Computer vision is a field of AI that enables machines to interpret and understand visual information from images and videos. Core tasks include image classification, object detection, semantic segmentation, and image generation.

- **Image classification** assigns a single label to an entire image based on its content (e.g., "cat" or "dog")
- **Object detection** identifies and localizes multiple objects within an image by drawing bounding boxes and assigning class labels to each detected object
- **Semantic segmentation** classifies every pixel in an image into a category, enabling precise boundary delineation
- **Feature extraction** uses CNNs to automatically learn hierarchical features from raw pixels -- edges and textures in early layers, complex shapes and objects in deeper layers
- **Popular architectures** include YOLO (real-time detection), ResNet, VGG, and Vision Transformers (ViT) for various vision tasks

**Practical relevance:** Computer vision is deployed in autonomous vehicles, medical imaging diagnostics, facial recognition, manufacturing quality control, augmented reality, and surveillance systems. It is one of the primary application domains for deep learning and CNNs, making it a core topic in ISA 400.

### CNNs

**Definition:** Convolutional Neural Networks (CNNs) are a class of deep neural networks specifically designed for processing grid-structured data like images. They use learnable filters (kernels) that slide across the input to automatically detect spatial features at multiple levels of abstraction.

- **Convolutional layers** apply sets of small filters to the input, detecting features like edges and textures in early layers and complex patterns (faces, objects) in deeper layers
- **Pooling layers** (max pooling, average pooling) downsample feature maps to reduce spatial dimensions, decrease computation, and provide translation invariance
- **ReLU activation** is applied after each convolution to introduce non-linearity, enabling the network to learn complex mappings
- **Fully connected layers** flatten the extracted features and pass them through dense layers for final classification via a softmax output
- **Typical architecture flow:** Input -> [Conv -> ReLU -> Pool] x N -> Flatten -> Fully Connected -> Output

**Practical relevance:** CNNs revolutionized computer vision and remain the backbone of image classification, object detection, medical imaging, and video analysis. Architectures like ResNet, VGG, and EfficientNet are widely used in production systems. Understanding CNNs is essential for implementing vision models in PyTorch.

### RNNs

**Definition:** Recurrent Neural Networks (RNNs) are a class of neural networks designed for sequential data where the order of elements matters. Unlike feedforward networks, RNNs maintain a hidden state that acts as memory, feeding the output from one time step back as input to the next.

- **Hidden state** is updated at each time step based on the current input and the previous hidden state, enabling the network to maintain context across a sequence
- **Vanishing gradient problem** is a key limitation where gradients diminish exponentially during backpropagation through long sequences, preventing the network from learning long-range dependencies
- **LSTM (Long Short-Term Memory)** addresses the vanishing gradient problem using a cell state and three gates (input, forget, output) that regulate information flow, enabling learning over long sequences
- **GRU (Gated Recurrent Unit)** is a simplified variant of LSTM with two gates (reset, update) that achieves similar performance with fewer parameters
- **Applications** include speech recognition, machine translation, text generation, time series forecasting, and handwriting recognition

**Practical relevance:** While transformers have largely replaced RNNs for many NLP tasks, RNNs and LSTMs remain important for time series analysis, real-time sequence processing, and understanding the evolution of sequence modeling in deep learning. They are a foundational architecture covered in ISA 400.
