### SQL injection

**Definition:** SQL injection (SQLi) is a web security vulnerability that allows an attacker to interfere with the queries an application makes to its database by injecting malicious SQL code, often leading to data theft, unauthorized access, or system compromise.

**Key Concepts:**
- **Types:** In-band SQLi (attacker sees results directly), Blind SQLi (infers information from application behavior), and UNION-based attacks (retrieves data from other database tables)
- **Impact:** Attackers can view, modify, or delete data belonging to other users, and in severe cases escalate to full system compromise
- **Prevention:** Use parameterized queries and prepared statements, implement allow-list input validation, enforce least-privilege database access, and avoid exposing database error messages to clients

**Practical Relevance:** SQL injection consistently ranks among the OWASP Top 10 vulnerabilities. Understanding SQLi is essential for any developer building database-driven web applications, as even a single unparameterized query can expose an entire database to attack.

### Cross-site scripting (XSS)

**Definition:** Cross-Site Scripting (XSS) is an injection attack where malicious scripts are injected into trusted websites, allowing attackers to send browser-side code to unsuspecting end users.

**Key Concepts:**
- **Reflected XSS:** Malicious script comes from the current HTTP request and is immediately reflected back to the user
- **Stored XSS:** Malicious script is permanently stored in the website's database and served to all visitors
- **DOM-based XSS:** Vulnerability exists in client-side JavaScript code rather than server-side code
- **Prevention:** Output encoding, HTML sanitization (e.g., DOMPurify), Content Security Policy (CSP) headers, HttpOnly cookie flags, and Trusted Types browser API

**Practical Relevance:** XSS attacks can lead to session hijacking, account takeover, phishing, and sensitive data exposure. OWASP recommends framework security protections, output encoding, and HTML sanitization as the best defenses for all web applications.

### Mobile security

**Definition:** Mobile security encompasses the strategies, technologies, and practices used to protect mobile devices, applications, and data from threats including malware, phishing, and unauthorized access.

**Key Concepts:**
- **Common Threats:** Malicious apps, phishing and SMiShing (SMS phishing), Man-in-the-Middle (MitM) attacks, mobile ransomware, spyware, and data leaks
- **Protection Strategies:** Regular OS updates, mobile security software, strong authentication (MFA), limiting app permissions, and employee cybersecurity training
- **Enterprise Solutions:** Mobile Device Management (MDM) for remote monitoring, encryption enforcement, and secure app deployment; zero-trust architectures requiring continuous verification of users and devices

**Practical Relevance:** As mobile devices increasingly handle sensitive business and personal data, mobile security is critical for both individual users and organizations. NIST and NSA publish mobile device best practices that guide enterprise security policies in the hybrid workforce era.

### Encryption

**Definition:** Encryption is the process of converting plaintext data into an unreadable ciphertext format using mathematical algorithms and keys, ensuring that only authorized parties can access the original information.

**Key Concepts:**
- **Symmetric Encryption:** Uses a single shared key for both encryption and decryption; faster and efficient for large data volumes (e.g., AES, DES, Blowfish)
- **Asymmetric Encryption:** Uses a public key to encrypt and a private key to decrypt; more secure but slower (e.g., RSA, ECC, Diffie-Hellman)
- **Hybrid Systems:** Modern systems combine both -- asymmetric encryption securely shares a symmetric key, which then encrypts the actual data
- **Use Cases:** Symmetric for file encryption, VPNs, and databases; asymmetric for digital signatures, SSL/TLS, secure email, and cryptocurrency

**Practical Relevance:** Encryption is the foundation of data confidentiality in cybersecurity. Understanding when to use symmetric versus asymmetric methods is essential for securing data at rest and in transit across networks, web applications, and cloud services.

### Virtual labs

**Definition:** Virtual cybersecurity labs are controlled, software-based environments that simulate real IT infrastructure and provide hands-on training through challenges and scenarios without risking production systems.

**Key Concepts:**
- **Safe Experimentation:** Mistakes in virtual labs do not impact live networks, data, or revenue, allowing risk-free learning and exploration
- **Real-World Relevance:** Labs replicate scenarios cybersecurity professionals face daily, building critical problem-solving skills through simulated threat response
- **Popular Platforms:** TryHackMe (browser-based gamified labs), Hack The Box, CyberRange, and cloud-based platforms like ACI Learning and Immersive Labs
- **Accessibility:** Virtual labs enable remote access to training environments from any location with an internet connection

**Practical Relevance:** With information security analyst roles projected to grow 33% from 2023 to 2033 (U.S. Bureau of Labor Statistics), virtual labs have become an essential tool for building practical cybersecurity skills that employers demand in a rapidly growing field.

### Threat analysis

**Definition:** Threat analysis is a structured cybersecurity process that formally evaluates the degree of threat to an information system by identifying, assessing, and prioritizing threats based on adversary behavior, attack techniques, and business impact.

**Key Concepts:**
- **Risk Formula:** Risk = Damage Potential (impact) x Likelihood (probability of the threat occurring)
- **Frameworks:** STRIDE (Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege), PASTA, OCTAVE, and the MITRE ATT&CK knowledge base
- **Assessment Types:** Quantitative (numerical data and statistical models) and qualitative (expert judgment and categorization) approaches
- **Process:** Identify assets, enumerate threats, assess vulnerabilities, calculate risk ratings, and develop mitigation strategies

**Practical Relevance:** Threat analysis is foundational to any organization's security posture. Frameworks like MITRE ATT&CK and methodologies like STRIDE help security teams systematically identify risks and allocate resources to the most critical vulnerabilities before attackers exploit them.

### Security vulnerabilities

**Definition:** A security vulnerability is a flaw or weakness in an application's design or implementation that allows an attacker to cause harm, ranging from data breaches to full system compromise.

**Key Concepts:**
- **OWASP Top 10 (2025):** The most critical web application risks include Broken Access Control (#1), Security Misconfiguration (#2), Cryptographic Failures (#3), Vulnerable Components (#4), and Injection (#5)
- **CVE System:** Common Vulnerabilities and Exposures is the global standard for identifying and cataloging known security vulnerabilities across all software products
- **CWE:** Common Weakness Enumeration classifies the types of software weaknesses that lead to vulnerabilities
- **Common Types:** SQL injection, XSS, broken access control, unencrypted data in transit, and security misconfiguration

**Practical Relevance:** The OWASP Top 10 serves as the industry-standard awareness document for developers and security professionals. Understanding CVE identifiers and CWE classifications enables teams to track, communicate about, and remediate vulnerabilities systematically across their software portfolio.
